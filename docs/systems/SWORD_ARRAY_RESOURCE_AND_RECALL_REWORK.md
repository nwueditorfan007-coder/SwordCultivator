# 剑阵资源与回收制重构方案

## 1. 文档定位

本文档用于定义剑阵系统下一阶段的资源重构方案。

本次重构的目标不是继续扩展“吸收冻结弹幕 -> 转化为可发射弹丸”的旧链路，而是将剑阵系统拆分为两套职责明确的资源：

- `剑意`：常规战斗资源，用于支撑御剑与剑阵输出
- `剑丸资源`：稀缺强化资源，用于主动开启短时强化剑技

同时，原先作为消耗物的“剑阵子弹”将改造为可见、可射出、可回收的 `剑阵飞剑` 池。

本文档将作为本轮系统改造与分步实现的直接依据。


## 2. 核心结论

本轮重构后，战斗循环调整为：

1. 普通攻击负责近身压制与弹反，不再负责截停和转化剑丸
2. 剑阵输出改为消耗 `剑意`
3. `剑丸资源` 改为稀缺掉落资源，只用于启动强化剑技
4. `剑阵飞剑` 改为常驻可见的战斗载体，并遵循“射出 -> 命中/到达终点 -> 回收 -> 重新挂阵”的周转规则
5. 环阵 / 扇阵 / 贯穿阵的区别，不再只是几何形态差异，还应包含单次射量、回收节奏、可持续输出能力的差异

换句话说：

- `剑丸资源` 不是弹药，而是爆发钥匙
- `剑阵飞剑` 不是掉落资源，而是武器库存
- `剑意` 不是强化货币，而是日常输出成本


## 3. 术语定案

为避免后续实现、UI 与讨论中混淆，本轮统一术语如下：

### 3.1 剑丸资源

- 一种稀缺强化资源
- 初始上限为 `3`
- 常规战斗中主要通过掉落获得
- 用途是主动释放强化剑技

### 3.2 剑阵飞剑

- 指围绕玩家、可被剑阵发射、命中后会回收的飞剑实体
- 默认可用数量为 `6`
- 强化剑技期间提升为 `12`
- 剑阵发射消耗的不是“资源数量”，而是当前可用飞剑库存与对应 `剑意`

### 3.3 剑意

- 常规战斗资源
- 主要通过近战、弹反、战斗过程积累
- 用于维持御剑与剑阵输出


## 4. 设计目标

本轮重构的设计目标有 5 个：

1. 降低战场语义复杂度，移除旧系统中“冻结弹幕 / 吸收 / 再发射 / 大小剑丸”叠在一起的理解成本
2. 保留并强化“御剑武器池”的题材识别度，让飞剑数量、射出状态、回收状态都在战场上清晰可见
3. 让三种剑阵在“爆发 / 轮替 / 持续”上形成明确节奏身份
4. 让强化剑技成为一个短时、高辨识度、可主动规划的爆发窗口
5. 让玩家主要通过战场观察，而不是数字 HUD，来感知剑阵当前可用量与冷却恢复情况


## 5. 资源层

## 5.1 总体结构

重构后，系统中存在三套彼此独立的资源：

- `生命`：生存资源
- `剑意`：常规输出资源
- `剑丸资源`：强化启动资源

其中：

- `剑阵飞剑` 是战斗实体，不归类为掉落资源
- `剑阵飞剑` 的周转状态会影响剑阵是否能继续发射
- `剑意` 与 `剑阵飞剑库存` 共同构成剑阵的实际开火条件

## 5.2 剑丸资源规则

- 初始上限：`3`
- 初始持有：建议首版为 `0` 或 `1`
- 获取方式：
  - 中等怪死亡时有概率掉落
  - Boss 战期间按机制或概率刷出
- 使用方式：
  - 玩家按 `Space` 主动消耗 `1` 个剑丸资源
  - 进入持续 `5s` 的强化剑技状态

不再允许以下旧行为：

- 通过主动吸收冻结弹幕获得剑丸
- 通过不同大小弹幕映射到不同剑丸规格
- 让剑丸资源同时兼任掉落物、弹药、强化层数三种身份

## 5.3 剑阵飞剑库存规则

- 普通状态下总库存：`6`
- 强化状态下总库存：`12`
- 每把飞剑都必须处于以下三态之一：
  - `ready`：已挂回剑阵，可立即发射
  - `outbound`：已射出，正在攻击
  - `returning`：已完成攻击，正在飞回玩家

只有 `ready` 状态的飞剑，才算当前可用飞剑。

## 5.4 剑阵开火条件

剑阵每次尝试发射时，必须同时满足：

1. 当前 `ready` 飞剑数量足够
2. 当前 `剑意` 足够支付本次发射成本
3. 当前输入状态允许对应阵型继续发射

如果飞剑不足，则剑阵进入“库存等待”状态。
如果剑意不足，则剑阵进入“能量不足”状态。

这两个状态在表现上应可区分，避免玩家误判。

进一步要求：

- 玩家尝试释放技能失败时，应立即收到反馈，而不是只能自己观察数值条
- `剑意不足` 与 `飞剑库存不足 / 正在回收` 应尽量用不同提示表现
- 若后续 `御剑`、`强化剑技` 也接入同类反馈，应一并区分 `剑丸不足`


## 6. 输入层

## 6.1 基础操作定案

- `左键点击`：普通攻击
- `左键长按`：启动并维持剑阵攻击
- `右键点按 / 长按`：御剑逻辑，延续现有点刺 / 连斩结构
- `Space`：消耗 `1` 个剑丸资源，主动开启强化剑技

## 6.2 普通攻击职责调整

普通攻击不再承担“截停剑丸 / 收集剑丸 / 转化剑丸”的职责。

普通攻击的新职责为：

- 近身伤害
- 防御性弹反
- 维持近战存在感
- 提供剑意收益

定案如下：

- 普通攻击命中敌方子弹时，应触发弹反
- 普通攻击击杀敌人，不再制造可吸收剑丸
- 飞剑击杀敌人，也不再制造可吸收剑丸

## 6.3 剑丸资源拾取规则

- 移除主动吸取
- 场上若生成 `剑丸资源掉落物`，玩家靠近后自动吸取
- 自动吸取应具有稳定半径与轻量吸附动画
- 若玩家资源已满，则掉落物可保留短时间或按系统规则消失


## 7. 剑阵飞剑系统

## 7.1 战场可见性原则

`剑阵飞剑` 必须是战场上持续可见的实体，而不是抽象库存数字。

玩家应通过观察战场，直接读懂以下信息：

- 玩家周身剑阵中还有几把 `ready` 飞剑
- 当前有多少把飞剑正在外飞攻击
- 当前有多少把飞剑正在回收
- 当前剑阵的恢复节奏是否快、是否慢

因此，本系统不依赖单独数字 HUD 告知库存，而应优先依赖实体可视化。

## 7.2 飞剑生命周期

每把飞剑的推荐生命周期如下：

1. `ready`
   - 挂在玩家周围的当前剑阵槽位上
   - 可参与当前阵型的视觉排布
   - 可被选为下一次发射源
2. `outbound`
   - 按当前阵型规则射出
   - 在战场上保持可见
   - 命中目标或飞至终点后结束攻击阶段
3. `returning`
   - 从当前位置飞回玩家
   - 在回收过程中仍然保持可见
   - 只有真正回到剑阵槽位后，才重新切回 `ready`

禁止以下行为：

- 回收途中提前计为可用
- 射出后直接凭空重置为 `ready`
- 为了保持阵型美观而隐藏库存缺口

## 7.3 剑阵缺口原则

若部分飞剑已射出或正在回收，玩家周围的剑阵应允许出现可见缺口。

这条规则的目的不是追求整齐，而是强化库存读感：

- 缺口越多，说明当前可立即发射的飞剑越少
- 回收飞剑逐步补回缺口，意味着冷却正在恢复

除非后续验证认为阅读性明显下降，否则不建议自动把剩余飞剑重新均匀铺满全阵来掩盖缺口。


## 8. 剑阵三形态节奏

## 8.1 总体原则

三种剑阵的差异，应该同时体现在：

- 几何形态
- 单次射量
- 飞剑库存周转方式
- 冷却读感
- 输出定位

核心目标是形成以下身份：

- 环阵：爆发清场
- 扇阵：轮替压制
- 贯穿阵：持续穿刺

## 8.2 环阵

设计定位：

- 近距离爆发
- 面对贴身压力时，快速清出安全区

发射规则建议：

- 单次发射时，尽可能一次性射出当前整批可用飞剑
- 普通态默认是 `6` 把同时射出
- 强化态默认是 `12` 把同时射出

节奏特征：

- 瞬时爆发最高
- 发射后玩家身边剑阵会大幅变空
- 必须等待整批飞剑回收，整体周转最慢
- 冷却感最明显

实现重点：

- 发射一瞬间要明显读出“全阵出鞘”
- 回收过程要足够清晰，否则玩家会误以为系统卡住

## 8.3 扇阵

设计定位：

- 中距离轮替压制
- 保持前方覆盖，同时兼顾节奏稳定

发射规则建议：

- 单次发射当前可用飞剑的一半
- 普通态默认分两波轮替
- 强化态也保持“分批轮替”身份，而不是简单复制环阵的全弹齐射

节奏特征：

- 单次爆发中等
- 可见地保留一部分飞剑继续挂阵
- 已射出飞剑回收前，另一半飞剑仍可支撑下一轮输出
- 冷却感明显弱于环阵

实现重点：

- 两波切换节奏要稳定，不应因为回收 timing 偶然重叠而读成乱射
- 前方扇面应始终看起来“还留着一半兵力”

## 8.4 贯穿阵

设计定位：

- 远距离持续穿刺
- 稳定输出高价值目标

发射规则建议：

- 单次尽量只射出 `1` 把或极少量飞剑
- 飞剑逐把周转，形成连续输出流
- 强化态主要提升飞剑池上限、射速与飞行速度，而不是改变其“逐把流转”的身份

节奏特征：

- 单发爆发最低
- 单位时间的出手频率最高
- 飞剑会持续处于“少量射出、少量回收、少量待命”的流动态
- 冷却读感最轻，但持续压制感最强

实现重点：

- 玩家要能明显看到飞剑一把一把地进出循环
- 不能因为回收过慢，让贯穿阵失去持续输出身份

## 8.5 阵型冷却定义

本系统中，剑阵的“冷却”不再由单独的固定 CD 决定，而主要由以下因素共同决定：

- 飞剑射出速度
- 飞剑到达终点或命中后的回收启动时机
- 飞剑回收飞行速度
- 该阵型单次射出的飞剑数量

因此：

- 环阵的冷却长，是因为单次清空库存更多
- 扇阵的冷却中等，是因为飞剑按批次轮替
- 贯穿阵的冷却短，是因为飞剑逐把流转


## 9. 强化剑技

## 9.1 启动规则

- 玩家按 `Space`
- 消耗 `1` 个剑丸资源
- 若当前资源足够，则进入 `5s` 强化剑技状态

若资源不足，则不触发。

## 9.2 强化效果定案

强化期间：

- 御剑时停持续更久
- 剑阵飞剑飞行速度更快
- 剑阵单次射量更多
- 剑阵飞剑总库存从 `6` 提升到 `12`

其中“单次射量更多”的具体实现需保持阵型身份一致：

- 环阵：仍然是整批齐射，但总数更多
- 扇阵：仍然是分批轮替，但每批数量更多
- 贯穿阵：仍然是逐把或小批连射，但周转更快、密度更高

## 9.3 强化的设计作用

强化剑技应承担以下体验目标：

- 主动制造爆发窗口
- 让玩家为关键战斗节点留资源
- 在 Boss 战中提供一个“准备好后主动进攻”的节奏峰值

它不应只是单纯的伤害倍数按钮，而应体现“御剑系统全面升压”的状态变化。

## 9.4 强化与剑意的关系

强化剑技不会替代 `剑意` 机制。

强化期间：

- 剑阵仍然消耗剑意
- 御剑仍然受剑意体系约束
- 强化只是让飞剑池、飞行速度、射量、时停收益进入更强状态

这样可以避免 `剑丸资源` 直接变成“第二套主输出货币”。


## 10. UI 表现

## 10.1 总体原则

UI 必须同时满足两种诉求：

1. 核心战斗状态优先通过战场实体可视化表达
2. 资源与强化状态仍然需要有稳定 HUD 辅助

## 10.2 战场内可视化

优先级最高的表现层应是战场内信息：

- 玩家周身可见 `ready` 飞剑
- 外飞中的 `outbound` 飞剑保持可见
- 回收中的 `returning` 飞剑保持可见
- 回到阵中的瞬间有明确的补阵反馈

玩家应主要通过这些信息判断：

- 当前还能不能继续发射
- 当前回收速度快不快
- 当前阵型是否进入空窗

## 10.3 剑意条

剑意条继续作为常规资源条存在。

强化期间建议增加明显高亮效果，例如：

- 条体边缘发光
- 火焰感流动
- 强化剩余时间的呼吸闪烁

设计目标是让玩家在不看文本的情况下，一眼知道自己正处于强化态。

## 10.4 剑丸资源显示

`剑丸资源` 建议使用独立图标显示，而不是并入剑意条。

推荐形式：

- 固定 `3` 格资源图标
- 有资源时点亮
- 消耗时有明显熄灭反馈
- 拾取时有短暂跳动或填充动画

## 10.5 强化状态显示

除剑意条高亮外，建议额外提供至少一种强化态提示：

- 玩家周身火焰/剑气强化环
- HUD 上出现强化计时环或倒计时条
- 剑阵飞剑本体颜色或拖尾增强

要求是：即使玩家不看资源区，也能感受到角色进入了“强化武装”状态。


## 11. 数值风险点

## 11.1 剑丸资源掉落波动

若剑丸资源完全依赖随机概率掉落，容易出现两个问题：

- 运气差时整局都无法稳定进入强化
- 运气好时强化窗口连续出现，导致强度波动过大

建议：

- 概率掉落外观下加入保底逻辑
- Boss 战至少保证阶段性稳定来源

## 11.2 环阵空窗过长

环阵若一次性射出全部飞剑，但回收路径或回收速度过慢，容易出现：

- 爆发一瞬很爽
- 后续空窗过长
- 玩家被迫切模式，不再愿意使用环阵

需重点盯防：

- 回收速度
- 回收起点距离
- 环阵在高压近战时的自保能力

## 11.3 扇阵轮替失稳

扇阵如果分批规则不稳定，可能出现：

- 两波节奏挤在一起
- 回收节奏与发射节奏互相打架
- 玩家无法预测下一批什么时候可用

因此扇阵的分批逻辑应尽量明确、稳定、可预期。

## 11.4 贯穿阵身份被强化态破坏

强化态如果只是一味堆单次射量，可能导致贯穿阵失去“逐把流转”的身份，变成另一种小环阵。

建议：

- 强化贯穿阵时，优先加强周转、飞行速度、持续密度
- 谨慎增加其单批齐射量

## 11.5 普通攻击存在感下降

移除“截停并吸收弹幕”后，普通攻击可能失去原本的一部分系统重量。

因此普通攻击改成弹反后，必须确保它仍然有明确收益，例如：

- 更稳定的剑意获取
- 更强的近身防御价值
- 弹反成功后的额外输出回报

## 11.6 剑意与飞剑库存双重卡手

如果剑意不足与飞剑不足同时频繁发生，玩家容易感到“什么都做不了”。

实现时要避免两个限制同时过严。

推荐原则：

- 平时主要卡一种
- 另一种作为次级约束


## 12. 分步实现建议

## 12.1 第一阶段：资源重构

先完成以下内容：

- 删除旧主动吸收逻辑
- 引入 `剑丸资源` 独立计数
- 引入场上掉落与自动拾取
- 将 `Space` 改为强化剑技启动键
- 将剑阵开火成本改为 `剑意`

此阶段先不追求完整回收视觉，只先把逻辑拆清楚。

## 12.2 第二阶段：飞剑池与回收状态

完成以下内容：

- 将剑阵飞剑从抽象弹道改为可追踪实体
- 建立 `ready / outbound / returning` 三态
- 让飞剑回到玩家周身后再恢复可用
- 让玩家周围阵型正确显示库存缺口

此阶段完成后，剑阵冷却将从“计时器感”切换为“实体回收感”。

## 12.3 第三阶段：三阵型节奏校准

完成以下内容：

- 环阵的整批齐射与长回收
- 扇阵的半批轮替

## 13. 当前定案总结

本轮重构后的系统方向是：

- 移除旧的“冻结弹幕 -> 主动吸收 -> 转化为发射弹丸”的主循环

- 将三种剑阵的区别扩展为“几何 + 节奏 + 周转”三层差异

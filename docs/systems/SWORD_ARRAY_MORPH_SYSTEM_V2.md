# 剑阵形变系统 V2 设计方案

## 1. 定位

V1 已完成三段剑阵、实时预览、发射前成阵与基础手感修正。  
V2 的任务不是推翻 V1，而是在保持当前战术分工清晰的前提下，把“三段硬切”升级为“连续形变”。

V2 应建立在以下前提上：

- 不增加新按键
- 不改变“准星距离决定剑势形态”的核心控制逻辑
- 不破坏 V1.5 已确认通过的发射节奏与命中可读性
- 先连续化视觉、排布与成阵过程，再评估是否连续化结算


## 2. V2 目标

### 2.1 核心目标

- 让剑阵在准星移动时呈现连续变形，而不是在阈值处突然切换
- 让玩家能直观看到“环阵张开为扇阵”“扇阵收束为贯穿阵”
- 保持三种形态的战术语义仍然清楚
- 让预览、待发排布、成阵动画共用同一套形变逻辑

### 2.2 非目标

V2 当前不作为定案范围的内容：

- 自动追踪
- 手动模式切换
- 剑丸融合升阶
- 伤害与碰撞体积的完全连续化
- 三种形态的机制特效强化


## 3. 设计原则

### 3.1 连续几何，离散语义

V2 的连续，主要体现在几何形状、排布和视觉反馈上。  
V2 的战术意义仍保持离散，即玩家仍能理解当前剑势主要偏向：

- 护体解围
- 中距压制
- 远程穿刺

### 3.2 预览优先

V2 第一优先级不是改伤害公式，而是保证玩家看到的剑阵变化可信、稳定、可读。  
如果连续预览成立，再考虑部分结算是否需要连续化。

### 3.3 发射逻辑稳于预览逻辑

预览和待发排布允许连续变化。  
发射节奏、批次和强度逻辑在 V2 阶段仍可沿用“主导形态”的离散规则，以降低风险。

### 3.4 同一套几何驱动全部表现

以下内容应尽量从同一套形变状态中求解：

- 预览图形
- 吸附中的剑丸站位
- 发射前收束/展开动画
- 发射方向求解的基础几何


## 4. V2 形变结构

### 4.1 距离分段

V2 不再只使用单点阈值，而是使用“稳定区 + 过渡区”。

建议距离区间如下：

- `0 - 120`：稳定环阵区
- `120 - 220`：环阵 -> 扇阵过渡区
- `220 - 360`：稳定扇阵区
- `360 - 500`：扇阵 -> 贯穿阵过渡区
- `500+`：稳定贯穿阵区

说明：

- 稳定区用于保证战术辨识度
- 过渡区用于提供连续形变空间
- 这组数值是 V2 首版建议值，后续可通过实机手感再调

### 4.2 主导形态

虽然 V2 会连续形变，但任意时刻仍应存在一个“主导形态”。

主导形态用于决定：

- 发射节奏
- 连发批次
- 主要特效风格
- 伤害与命中规则

建议规则：

- 在稳定区内，主导形态固定为该区形态
- 在过渡区内，主导形态按过渡进度是否越过中点判定
- 一旦长按达到发射阈值并开始正式发射，本轮持续发射锁定主导形态，直到松开左键

这样可以避免以下问题：

- 准星在过渡区来回抖动，导致节奏频繁跳变
- 预览是连续的，但实际连发逻辑在一秒内反复切换

### 4.3 形变状态

V2 建议引入统一的形变状态，而不是只返回字符串模式。

建议状态字段：

- `dominant_mode`：当前主导形态
- `visual_from_mode`：当前视觉起点形态
- `visual_to_mode`：当前视觉终点形态
- `visual_blend`：当前视觉混合进度，范围 `0.0 - 1.0`
- `distance_ratio`：准星距离的归一化结果

建议使用平滑曲线而不是线性硬插值：

- 过渡区内部使用 `smoothstep`
- 稳定区直接返回 `0.0` 或 `1.0`


## 5. 两段连续形变定义

### 5.1 环阵 -> 扇阵

目标体验：

- 从“围绕自身的护体圆环”逐渐打开成“朝前展开的压制扇面”

形变规则：

- 后半圈的视觉权重逐渐减弱，前半圈权重增强
- 原本均匀环绕的剑丸逐渐向鼠标朝向前方偏移
- 圆环的闭合感逐渐减少，前方开口和扇面边界逐渐形成
- 待发站位从“全周均分”过渡到“前方分层扇面”
- 预览从圆环双层线框逐渐过渡为扇形带状区域

需要强调的感受：

- 这一段不是“圆突然变扇”
- 而是“护体剑势被玩家向前牵开”

### 5.2 扇阵 -> 贯穿阵

目标体验：

- 从“前方扇面压制”逐渐收束为“狭长直线穿刺”

形变规则：

- 扇面弧度逐渐收窄
- 左右两翼的站位逐渐向中轴靠拢
- 外缘半径逐渐向前延长，形成明显前探感
- 扇面填充感逐渐减弱，中心脊线与尖端提示逐渐强化
- 待发站位从“多层扇面”过渡到“窄线/楔形队列”
- 预览从扇形区域逐渐过渡为细长剑轨与前端尖点

需要强调的感受：

- 这一段不是“扇面缩小”
- 而是“压制面被收束成刺穿力”


## 6. 表现层定案

### 6.1 预览表现

预览必须始终是 V2 最先完成、最稳定的一层。

具体要求：

- 玩家在未发射前即可明确看见形变方向
- 在过渡区移动准星时，预览应连续变化，不出现跳帧式切换
- 过渡中仍应保留主导形态的识别锚点

识别锚点建议：

- 偏环阵时保留明显的近身环感
- 偏扇阵时保留明确的扇面边界
- 偏贯穿阵时保留清晰的中心线与尖端

### 6.2 吸附剑丸排布

吸附在玩家周围的剑丸不应只在发射瞬间切换形态。  
当玩家拖动准星时，剑丸排布本身就应呈现连续变形。

具体要求：

- 站位求解与预览使用同一套形变状态
- 过渡区内每颗剑丸应有稳定目标位，避免来回抖动
- 当吸附数量较少时，中间态也要保持可读，不因数量不足而完全失真

### 6.3 发射前成阵

V1.5 已经有基础成阵收束动画。  
V2 要求成阵也体现连续形变，而不是先切模式再播放对应动画。

具体要求：

- 长按初段先按当前形变状态收束
- 如果准星仍在移动，收束中的形态允许平滑跟随
- 一旦进入正式发射阶段，本轮主导形态锁定，但基础朝向仍可跟随鼠标更新


## 7. 结算层策略

### 7.1 V2 当前定案

V2 先不追求伤害结算完全连续化。

当前推荐策略：

- 视觉与排布：连续
- 发射方向基础几何：尽量连续
- 发射节奏与批次：按主导形态离散
- 伤害与碰撞：按主导形态离散

### 7.2 原因

这样处理的原因是：

- V1.5 的环阵、扇阵、贯穿阵已经具备明确战术定位
- 若在 V2 首版同时连续化视觉和结算，调试成本会显著增加
- 玩家首先感知的是形态变化是否顺、预览是否可信，而不是过渡态伤害是否绝对连续

### 7.3 后续可评估项

如果 V2 实机手感稳定，可在后续版本评估：

- 过渡区弹道散布是否做部分连续化
- 扇阵向贯穿阵过渡时，是否让弹道集中度随进度提升
- 环阵向扇阵过渡时，是否让近身覆盖权重随进度衰减


## 8. 工程落地建议

### 8.1 配置层

建议在配置层新增：

- 距离稳定区与过渡区参数
- 过渡区平滑曲线参数
- 环扇过渡与扇贯穿过渡的独立插值系数

配置职责：

- 只描述数值和区间
- 不直接写形变逻辑

### 8.2 控制器层

控制器建议新增统一接口，例如：

- 根据准星距离返回 `morph_state`
- 根据 `morph_state` 计算预览数据
- 根据 `morph_state` 计算吸附剑丸站位
- 根据 `dominant_mode` 返回发射节奏与批次规则

控制器职责：

- 承担几何求解与模式归约
- 不在渲染层重复计算形变逻辑

### 8.3 渲染层

渲染层应只消费 `morph_state` 与预览数据，不自行判定模式切换。

渲染职责：

- 呈现连续预览
- 呈现连续成阵反馈
- 保留主导形态的识别特征


## 9. 验收标准

V2 首版通过的标准如下：

- 在原 V1 阈值附近来回移动准星时，不再出现明显“硬切”
- 玩家能肉眼看出环阵向扇阵、扇阵向贯穿阵的连续变化
- 预览、吸附排布、发射前成阵三者变化方向一致
- 正式发射后，节奏与手感不因过渡区抖动而频繁跳变
- 过渡态虽然连续，但仍能辨认当前主要偏向哪种战术用途


## 10. 测试重点

### 10.1 可读性测试

- 少量剑丸时是否还能识别形变方向
- 大量剑丸时是否会显得过于拥挤
- 中间态是否会出现既不像环也不像扇、既不像扇也不像贯穿的失真阶段

### 10.2 手感测试

- 在过渡区边缘小幅拖动准星时，是否感觉稳定
- 长按成阵到发射的过程是否顺滑
- 发射中改变准星方向时，是否既保持操控感又不导致节奏混乱

### 10.3 对齐测试

- 预览几何是否仍与实际发射方向一致
- 过渡态下的站位与预览是否一致
- 不同吸附数量下是否仍保持基本一致性


## 11. 与后续版本的关系

V2 的职责是建立“连续形变骨架”。  
后续版本可以在这个骨架上继续叠加：

- V2.5：形态专属战术价值强化
- V3：剑丸融合与高阶剑阵

因此 V2 的实现重点不是做满，而是打好一个稳定、可扩展、可调参的基础。

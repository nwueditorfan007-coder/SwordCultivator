# 剑势余韵与时停合式设计

- 日期：`2026-04-28`
- 性质：系统设计准定稿
- 状态：作为下一阶段原型实现与手感验证依据
- 适用范围：`剑势余韵`、`剑阵合式`、`御剑时停合式`、`剑鸣潮`、后续 `剑技领悟`

## 1. 这份文档解决什么问题

当前剑阵系统已经具备三个重要基础：

- 鼠标距离驱动的连续剑阵变形
- 环阵 / 扇阵 / 贯穿阵的职责分化
- 右键御剑时停带来的专注布剑窗口

但当前体验还缺少一个让玩家上头的短循环。

玩家可以判断当前该用哪个阵，但还没有形成足够强的下一步期待：

> 我刚才用某一式打出了势，下一招可以借这个势打出更帅、更强、更有形态变化的一击。

因此本系统的目标不是额外堆伤害，也不是增加战场标记，而是建立一套轻量的 `A + B` 合式语法：

- `A`：上一段战斗中形成的 `剑势余韵`
- `B`：下一次剑阵或御剑时停
- 结果：下一式带出上一式的影子，形成短时强化形态

## 2. 核心设计结论

本轮定案如下：

1. 放弃 `剑痕` 作为核心包装
2. 不引入 `阵眼` 作为独立空间系统
3. 使用 `剑势余韵` 承担剑阵之间、剑阵与御剑时停之间的联动
4. 剑阵触发余韵时，表现为 `上一式的势` 强化 `当前阵型`
5. 御剑时停触发余韵时，表现为 `上一式的影子` 短暂显形
6. 同时只保留一种主余韵，避免状态管理过重
7. `危险押注型升级` 不混入基础升级，后续独立为 `剑技领悟` 天赋树
8. `剑势爆发` 应与关卡强相关，进入 `剑鸣潮` 时刷出更多可收割小怪

这套系统的核心体验是：

> 立势 -> 得余韵 -> 接合式 -> 触发短时形态 -> 在剑鸣潮中放大收益。

## 3. 设计目标

### 3.1 增加上头感

玩家要产生短期追求：

- 再维持一下环阵，就能拿到环式余韵
- 再用扇阵连斩几个，就能接时停分光
- 带着刺式余韵进时停，就能画出一条剑虹贯日

上头点来自下一招的期待，而不是单纯数值变大。

### 3.2 强化连续变阵

余韵系统应该鼓励玩家轮转阵型，而不是固定一个阵型打到底。

玩家不是只问：

> 现在哪个阵最强？

而是开始问：

> 我现在用这个阵，是不是能给下一式起势？

### 3.3 保持战场可读性

本系统不在地面留下大量线条，不给敌人挂复杂标记，不新增常驻战场图层。

信息主要放在：

- 玩家身边或 UI 上的余韵提示
- 下一式触发时的短时形态变化
- 右键时停期间的极简预览

### 3.4 不脱离游戏核心

余韵不应该变成外置技能系统。

它必须始终服务现有核心：

- 环阵：近身可靠、防护、消弹、解压
- 扇阵：中距离面扫、覆盖、清群
- 贯穿阵：远距离破线、贯穿、处理高价值目标
- 御剑时停：让玩家专注御剑位置与下一次布剑判断

## 4. 术语

### 4.1 剑势余韵

玩家通过持续维持某个剑阵，或用某个剑阵打出连续击杀后，获得的短暂势能。

余韵不是资源条，也不是敌人状态。它更像上一式未散的势。

首版只保留三类：

- `环式余韵`
- `扇式余韵`
- `刺式余韵`

### 4.2 剑阵合式

玩家带着某种余韵，使用另一个剑阵时触发的强化形态。

剑阵合式的原则是：

> 上一式提供势，当前式决定形。

例如：

- `环式余韵 + 贯穿阵`：环势收束成一线，强化当前贯穿
- `扇式余韵 + 贯穿阵`：扇势分光，强化当前贯穿为多线

### 4.3 御剑时停合式

玩家带着某种余韵，触发右键御剑时停时出现的短时御剑形态。

御剑时停合式的原则是：

> 上一式的影子在时停中显形。

例如：

- `环式余韵 + 时停`：护身剑环短时显形
- `扇式余韵 + 时停`：御剑短时分出多把影剑
- `刺式余韵 + 时停`：完全时停中画下剑虹轨迹

### 4.4 剑鸣潮

剑势爆发的关卡版本。

剑鸣潮不是普通玩家 buff，而是关卡被剑势点燃后，短时间刷出更多可收割小怪，让玩家用合式和余韵打出连锁爽点。

### 4.5 剑技领悟

后续独立天赋树系统，用于承载危险押注型升级。

它不属于首版余韵实现范围，但应为余韵、剑鸣潮和高风险玩法提供长期构筑方向。

## 5. 余韵获得规则

余韵获得应来自两类行为。

### 5.1 持续立势

玩家持续维持某一阵型一段时间后，获得对应余韵。

设计目的：

- 给新手稳定获得余韵的路径
- 奖励玩家在正确场景中坚持某个阵型
- 让环阵这种偏稳场的阵型也能稳定起势

建议初值：

- 持续维持同一发射形态 `1.2s - 1.8s` 后获得余韵
- 期间允许短暂距离抖动，但发射判定形态不能频繁切换
- 获得余韵后继续维持同阵型，只刷新持续时间，不叠层

### 5.2 连斩顺势

玩家用某一阵型在短时间内击杀多个敌人后，获得对应余韵。

设计目的：

- 给熟练玩家更快的起势路径
- 让剑鸣潮中的大量小怪可以反哺合式循环
- 让上头感来自表现，而不是等待计时器

建议初值：

- 同一阵型在 `2.0s - 3.0s` 内击杀 `3` 个普通敌人，获得对应余韵
- 精英或关键目标可按更高权重计算
- 连斩计数应归属实际造成击杀的阵型

### 5.3 保留与消耗

首版建议规则：

- 同时只保留一种余韵
- 新余韵会覆盖旧余韵
- 余韵持续 `4s - 6s`
- 使用剑阵合式或御剑时停合式后消耗余韵
- 同阵型继续作战只刷新余韵，不触发合式

这样能避免玩家同时管理多个状态，也能强化 `A + B` 的清晰度。

## 6. 剑阵合式设计

剑阵合式的表现重点不是召唤额外系统，而是让当前阵型带出上一式的势。

首版可以先实现少量高辨识度组合，但完整设计先记录如下。

### 6.1 环式余韵 + 贯穿阵：圆势归一

形态：

- 贯穿阵起手时，飞剑先短暂呈圆弧收束
- 环式余韵像被压成一条锋线
- 最终以更凝聚、更清楚的一线刺出

战术作用：

- 用环阵稳住近身后，转为贯穿破线
- 适合打精英、护盾、后排术士、Boss 硬直
- 不是扩大清群范围，而是把近身稳场转化为一次高确定性的线性突破

设计理念：

> 环阵解决活下来，贯穿阵兑现破点。

### 6.2 环式余韵 + 扇阵：开莲外扫

形态：

- 扇阵起手不直接从前方展开
- 先从玩家周身出现一圈短暂剑影
- 剑影向外打开成扇面，像护身环被展开成莲瓣

战术作用：

- 从被包围状态转入中距离清场
- 适合玩家刚用环阵挡住贴脸压力后，把周围敌人推出去
- 让扇阵获得更安全的起手，但不让扇阵替代环阵的长期护身职责

设计理念：

> 环势开莲，把防守转成外扫。

### 6.3 扇式余韵 + 贯穿阵：分光穿刺

形态：

- 贯穿阵不只射出一条主线
- 沿扇阵残留的势分出 `2 - 3` 条平行或微展开的细剑线
- 主线仍最清楚，分线作为短时影刃

战术作用：

- 适合处理中距离成排敌人
- 可以穿过小怪同时打到后排目标
- 比普通贯穿更适合清一组，但覆盖仍小于扇阵

设计理念：

> 扇势未散，化作分光。

### 6.4 扇式余韵 + 环阵：收扇护体

形态：

- 环阵起手时，几道扇形剑影向玩家身边折回
- 折回后合成短时护身环
- 视觉上像扇面被收拢成一圈

战术作用：

- 用扇阵清了一侧后，快速回到近身防守
- 适合敌人从侧面推进后突然贴脸的情况
- 强化环阵起手可靠性，而不是增加远程输出

设计理念：

> 扇阵负责打开空间，环阵负责把空间守住。

### 6.5 刺式余韵 + 环阵：回锋成轮

形态：

- 环阵启动时，远端刺势像被拉回玩家身边
- 回锋过程中形成一次短促旋切
- 回到身边后进入正常环阵

战术作用：

- 贯穿阵打远处后，玩家可以借回锋保护自己
- 适合远程破线后被近身敌人补位的场景
- 让刺阵到环阵的切换有更强安全感

设计理念：

> 一剑出远，回锋护身。

### 6.6 刺式余韵 + 扇阵：裂线开面

形态：

- 扇阵起手时，先出现一条清楚的直线剑势
- 随后从这条线向两侧展开扇面
- 像一条刺线被横向撕开成面

战术作用：

- 适合先用刺阵建立方向，再用扇阵处理横向推进
- 对直线后方和中距离侧翼都有短暂覆盖
- 不应强过扇阵本身的持续面扫，只强化起手瞬间

设计理念：

> 刺阵定方向，扇阵开战线。

## 7. 御剑时停合式设计

御剑时停合式应明显不同于剑阵合式。

剑阵合式强化当前阵型。  
时停合式让上一式余韵在御剑中短时显形。

### 7.1 环式余韵 + 时停：护心剑环

形态：

- 触发御剑时停时，玩家周身出现短时旋转剑环
- 剑环自动围绕玩家旋转攻击
- 剑环可清理近身小怪和低威胁弹幕
- 玩家仍然可以把注意力放到御剑位置上

战术作用：

- 给玩家一个安全布剑窗口
- 解决右键时停时本体被贴脸干扰的问题
- 强化环阵的核心职责：近身可靠、防护、解压

设计理念：

> 环式余韵进入时停后，不是增加伤害，而是替玩家守住身体。

实现边界：

- 持续时间短，只覆盖时停前后的小窗口
- 不应长期替代环阵本体
- 不应清掉高威胁机制弹幕或 Boss 关键招

### 7.2 扇式余韵 + 时停：分光御剑

形态：

- 御剑短时间分出 `2 - 3` 把影剑
- 影剑跟随同一个鼠标意图，不增加额外操作对象
- 可以是平行分光，也可以是轻微扇面展开
- 主剑必须始终最清楚，影剑是辅助形态

战术作用：

- 提升御剑对一组敌人的覆盖
- 适合剑鸣潮中的小怪群
- 适合处理多个低血目标或多个可切断机制点

设计理念：

> 扇式余韵进入时停后，把单剑御剑暂时变成分光扫掠。

实现边界：

- 不允许玩家分别操控多把剑
- 影剑命中反馈应轻于主剑
- 影剑数量不宜太多，首版建议 `2` 把影剑或 `1 主 2 影`

### 7.3 刺式余韵 + 时停：剑虹贯日

形态：

- 刺式余韵触发御剑时停时，时停效果略微增强
- 出鞘的一小段时间内，时间完全静止
- 完全静止期间，玩家用鼠标在战场上画下一道轨迹线
- 完全时停结束后，飞剑沿轨迹高速穿透
- 执行完轨迹后，飞剑重新回到正常御剑跟随状态

这不是单纯的贯穿伤害提高，而是把刺阵的线性破点升级成玩家亲手画下的一道剑虹。

战术作用：

- 玩家可以绕过前排，画线穿过后排术士或关键目标
- 玩家可以在剑鸣潮中沿小怪路线画一条收割线
- 玩家可以在 Boss 破绽窗口中画出穿过弱点的最佳路径
- 玩家必须选择一条关键路线，而不是全屏乱清

设计理念：

> 刺式余韵进入时停后，玩家不只是瞄准一条线，而是在静止时间里写下一剑。

首版限制：

- 只限制 `画线时间`
- 只限制 `轨迹长度`
- 暂不限制转向角度
- 暂不做复杂路径合法性判断

建议初值：

- 完全时停画线时间：`0.6s - 0.9s`
- 最大轨迹长度：按当前屏幕与御剑尺度调参，首版以“能穿过一组敌人但不能绕完整场”为准
- 执行速度：必须明显快于普通御剑跟随，让玩家读成“一剑斩过”

可读性要求：

- 画线期间显示一条清楚但不刺眼的剑虹预览
- 轨迹到达最大长度时，预览末端应有明显停止反馈
- 飞剑执行轨迹时，主剑本体必须可读，不能只剩拖尾

## 8. 剑鸣潮设计

剑鸣潮是剑势爆发的关卡化版本。

它不应只是玩家获得一个短 buff，而应表现为：

> 玩家剑势已成，战场被剑鸣惊动，小怪被吸引涌入，玩家进入一段高密度收割窗口。

### 8.1 触发逻辑

建议规则：

- 玩家通过合式、连斩、正确阵型行为积累剑势
- 剑势满后进入 `剑鸣待发`
- 玩家主动触发剑鸣潮
- 触发后关卡导演刷出一波额外小怪

剑鸣潮最好不要自动触发，避免从奖励变成事故。

### 8.2 小怪规则

剑鸣潮刷出的敌人应服务收割爽点。

建议：

- 血量低
- 数量多
- 路线清楚
- 不直接刷在玩家脸上
- 不混入太多高威胁远程敌人
- 掉落或返还以剑意、剑势、连斩收益为主

### 8.3 与余韵的关系

剑鸣潮期间，余韵系统应该更容易滚动起来：

- 连斩更容易获得余韵
- 合式击杀可返还少量剑意
- 合式连杀可延长剑鸣潮少量时间
- 刺式余韵 + 时停的剑虹贯日可以成为剑鸣潮中的高光收割手段

这样剑鸣潮不是“刷更多怪给玩家砍”，而是：

> 关卡给玩家更多目标，玩家用余韵合式把目标串起来。

## 9. 剑技领悟设计边界

危险押注型升级不建议混在普通升级里。

它应作为独立系统：`剑技领悟`。

### 9.1 设计定位

普通升级负责稳定成长。  
剑技领悟负责改变玩法规则和承载风险收益。

### 9.2 推荐天赋方向

#### 守剑

偏稳定和生存。

例子：

- 环式余韵持续时间增加
- 护心剑环持续时间增加
- 剑意低于阈值时，环阵消耗降低
- 时停开始时短暂清理近身低级弹幕

#### 御剑

偏操作、布剑和合式。

例子：

- 余韵可保留时间增加
- 剑阵合式触发后，飞剑回收速度增加
- 分光御剑影剑数量增加，但影剑伤害降低
- 剑虹贯日最大轨迹长度增加

#### 狂剑

偏剑鸣潮与高风险收益。

例子：

- 剑鸣潮刷怪数量增加，但持续时间缩短
- 合式击杀返还更多剑意，但自然恢复降低
- 剑鸣潮期间受伤会缩短持续时间，但连杀可显著延长
- 开启剑鸣潮时召来精英影身，击杀后获得额外领悟收益

## 10. 首版实现建议

为了快速验证手感，首版不建议一次实现完整大矩阵。

### 10.1 必做

1. 余韵状态
   - 支持 `环式余韵 / 扇式余韵 / 刺式余韵`
   - 同时只保留一种
   - 支持持续立势和连斩顺势两种获得方式

2. 余韵提示
   - 玩家身边或 HUD 显示当前余韵
   - 获得时有短促音效与轻量视觉确认
   - 快消失时轻微闪烁或淡出

3. 三个高优先级合式
   - `环式余韵 + 贯穿阵：圆势归一`
   - `扇式余韵 + 时停：分光御剑`
   - `刺式余韵 + 时停：剑虹贯日`

这三个组合分别验证：

- 剑阵合式是否能强化当前阵型
- 时停显形是否能让余韵有形态变化
- 画线式御剑是否能成为刺余韵的标志性上头点

### 10.2 次优先

- `环式余韵 + 时停：护心剑环`
- `扇式余韵 + 贯穿阵：分光穿刺`
- `刺式余韵 + 环阵：回锋成轮`

### 10.3 后续

- 完整六个剑阵合式
- 剑鸣潮关卡导演
- 剑技领悟天赋树
- 余韵相关调试面板

## 11. 验证标准

首版实现后，应重点验证以下问题。

### 11.1 是否上头

玩家是否会主动说出类似体验：

- 我想再攒一次环余韵接贯穿
- 我想带扇余韵进时停清一波
- 我想用刺余韵画一条更好的剑虹线

如果玩家只觉得“伤害变高了”，说明形态变化还不够。

### 11.2 是否清楚

玩家是否能在不读长说明的情况下理解：

- 我什么时候获得了余韵
- 我现在持有什么余韵
- 为什么下一式变形了
- 哪个组合适合当前战况

如果玩家需要频繁看文字说明，说明反馈失败。

### 11.3 是否复杂

首版复杂度应控制在：

- 不新增按键
- 不新增常驻战场标记
- 不允许多个余韵并存
- 不要求玩家管理额外资源条

如果玩家开始觉得像在背系统表，说明组合数量或提示方式过重。

### 11.4 是否脱离核心

余韵系统必须让玩家更愿意使用环 / 扇 / 贯穿 / 御剑时停，而不是绕开它们。

如果某个合式变成万能答案，需要优先削它的覆盖或触发频率，而不是简单削伤害。

## 12. 当前设计判断

这套系统值得推进，原因是它正好补上当前玩法缺少的上头点：

- 它给玩家短期目标：攒余韵
- 它给玩家下一招期待：接合式
- 它给玩家形态反馈：上一式影子显形
- 它给关卡爆点：剑鸣潮中用合式收割
- 它不依赖更密弹幕或更乱画面来制造刺激

最重要的是，它没有脱离项目核心。

它不是外置技能树，不是地面陷阱，也不是通用状态异常。  
它是在当前剑阵与御剑时停之间，加上一层清楚的动作语法：

> 上一式成势，下一式借势。

# 时停特效正式节点化方案 V1

- 日期：`2026-04-25`
- 性质：实现导向设计文档
- 目标：为“御剑时停”建立一套符合正式特效生产流程的 Godot 节点化方案，使时停不再依赖 `main.gd` / `game_renderer.gd` 直接绘制最终效果，而是改为“代码驱动时机 + 节点负责视觉”。

## 1. 这份方案解决什么问题

当前项目围绕时停已经积累了三类东西：

- 正确的目标审美方向
- 一批可用的水墨贴图素材
- 一套能跑的程序化占位实现

但当前落地方式仍然偏“原型工程”：

- 主要视觉仍由代码直接决定
- 动态节奏没有被稳定地拆成时间轨道
- 素材的身份感很容易被程序图形吃掉
- 粒子、节点、动画、shader 还没有被明确分工

这会带来几个典型问题：

- 参数拉大之后只会更乱，不会更像目标图
- 美术很难在编辑器里直观调整
- 某一层做得不对时，很难快速判断是“素材错”“节奏错”还是“层级错”
- 后续要继续做 Boss 时停、关卡时停、镜头强化时，会重复踩同样的问题

这份文档的目标不是讨论“哪一张 crack 更好看”，而是先建立一套正式做法：

- 哪些效果必须交给 shader
- 哪些效果必须交给贴图节点
- 哪些效果必须交给粒子
- 哪些效果必须交给动画时间线
- 代码只该负责哪些驱动数据


## 2. 核心判断

### 2.1 时停是战场状态，不是单一贴图

时停的主感受来自三件事同时成立：

- 战场整体被冻结
- 飞剑方向切开画面
- 读屏主体仍然清楚

因此时停绝不能只靠：

- 一张全屏灰滤镜
- 一张 crack 贴图
- 一次性粒子爆点

它必须是一套分层系统。

### 2.2 代码不应该直接画最终造型

代码应该负责：

- 什么时候进入时停
- 焦点位置在哪里
- 方向向量是什么
- 当前处于进入 / 维持 / 恢复哪一段
- 当前强度是多少

代码不应该主要负责：

- 手工画裂墨几何
- 手工拼大笔触造型
- 用公式代替粒子层
- 用 `_draw()` 代替动画时间线

专业做法里，代码是“驱动器”，不是“画师”。

### 2.3 节点化不是为了更复杂，而是为了更可控

时停属于强演出状态层，适合节点化的原因不是“炫技”，而是：

- 可以在编辑器里单独预览每层
- 可以让美术直接调层级、透明度、时序
- 可以明确谁负责入场，谁负责维持，谁负责恢复
- 可以把素材真正作为主表达，而不是只做程序效果的调料


## 3. 正式做法总览

Godot 里这类时停效果的正式做法，建议拆成四类组件：

### 3.1 `Shader`

职责：

- 负责全屏状态变化
- 负责去色、对比、暗角、保色蒙版、轻微纸感

它负责的是：

- “世界被冻结了”

它不负责的是：

- 裂墨造型
- 飞剑方向声明
- 墨滴动态主节奏

### 3.2 `Sprite2D / TextureRect / Line2D`

职责：

- 负责可辨识的视觉形状

它负责的是：

- 宣纸底纹
- 大笔触
- 墨斑
- 裂墨
- 剑光 streak

这层决定：

- 玩家能不能一眼认出你的素材

### 3.3 `GPUParticles2D`

职责：

- 负责动态粒子和细碎喷溅

它负责的是：

- 裂墨飞溅
- 墨滴回流
- 冷雾细粒
- 入场爆发感

这层决定：

- 画面有没有“时停进入”的动势

### 3.4 `AnimationPlayer / Tween`

职责：

- 负责时间节奏

它负责的是：

- 什么时候爆
- 爆多久
- 什么时候冻结下来
- 什么时候开始恢复

没有这一层，效果就只会是“有层”，不会是“有演出”。


## 4. 推荐节点树

建议独立一个场景：

```text
scenes/vfx/TimeStopFx.tscn
```

推荐节点结构：

```text
TimeStopFx (CanvasLayer or Control)
- AnimationPlayer
- StateDriver
- FullscreenWash
  - WashRect (ColorRect + shader)
- BackgroundPaper
  - PaperBase (TextureRect)
  - PaperFiber (TextureRect)
- BackgroundBrush
  - BrushA (Sprite2D)
  - BrushB (Sprite2D)
  - BrushC (Sprite2D)
- MistLayer
  - MistNear (Sprite2D or GPUParticles2D)
  - MistFar (Sprite2D or GPUParticles2D)
- TargetLayer
  - TargetBlotPool (Node2D)
    - BlotDark_*
    - BlotLight_*
- EntryLayer
  - CrackDark (Sprite2D)
  - CrackLight (Sprite2D)
  - SwordStreakMain (Sprite2D or Line2D)
  - SwordStreakCore (Sprite2D or Line2D)
  - DropletBurst (GPUParticles2D)
  - DustBurst (GPUParticles2D)
- SustainLayer
  - FreezeHaloPool
  - SlowInkDrift (GPUParticles2D)
- DebugLayer
  - Optional gizmos / labels
```

### 4.1 为什么推荐 `CanvasLayer`

时停有明显的屏幕级特征：

- 要稳定覆盖战场
- 不受世界坐标抖动影响
- 更适合做 UI 不受污染的分层

如果后面要做更高级的背景专属处理，再升级为：

- `SubViewport + ColorRect + shader`

但第一版正式化，`CanvasLayer` 足够。


## 5. 分层职责

### 5.1 `FullscreenWash`

职责：

- 把战场从常态压到冻结态

内容：

- 去色
- 对比提升
- 轻纸感
- 轻暗角
- 局部保色

要求：

- 不能吃掉飞剑和目标
- 不能让 UI 脏掉
- 不能承担主造型

建议强度：

- 持续期常驻
- 进入时只做加速，不做唯一主表现

### 5.2 `BackgroundPaper`

职责：

- 让背景像宣纸，而不是灰滤镜

内容：

- 一层大纸纹
- 一层弱纤维

要求：

- 极轻
- 只能改变材质气质，不能改变构图

### 5.3 `BackgroundBrush`

职责：

- 在背景上建立少量水墨方向感

内容：

- 2 到 4 张低频笔触
- 沿飞剑主方向偏移

要求：

- 只属于背景
- 不可抢主体
- 不可变成几条巨大的黑色书法

### 5.4 `TargetLayer`

职责：

- 让目标读得更稳

内容：

- 敌人背后墨斑
- 轻微冻结边缘

要求：

- 只是“托住目标”
- 不是把敌人包进一团黑墨

### 5.5 `EntryLayer`

职责：

- 负责“时停进入”的主声明

内容：

- 沿飞剑方向的裂墨
- 白芯纸裂
- 墨滴爆发
- 剑光切开感

这是整个时停最重要的一层。

如果这一层不成立，玩家不会觉得“世界被剑撕开了”，只会觉得“画面变灰了”。

### 5.6 `SustainLayer`

职责：

- 把进入后的冲击感过渡成稳定的冻结态

内容：

- 轻雾
- 极慢的墨尘
- 目标边缘轻呼吸

要求：

- 不能比入场更抢
- 只是“状态存在”


## 6. 三段时间结构

时停必须拆成三段，而不是一个统一强度值。

### 6.1 进入 `Enter`

建议时长：

- `0.18s ~ 0.32s`

主层：

- `EntryLayer`
- `FullscreenWash` 快速增强

节奏：

1. 先有飞剑方向性的主切口
2. 裂墨和纸裂沿方向展开
3. 墨滴向外喷
4. 背景同时瞬间失色

注意：

- 入场要有“抢拍感”
- 不能只是平滑淡入

### 6.2 维持 `Hold`

建议时长：

- 跟玩法时停强度同步

主层：

- `FullscreenWash`
- `BackgroundPaper`
- `MistLayer`
- `TargetLayer`

节奏：

- 进入强冲击之后快速收静
- 世界保持冻结
- 飞剑和目标持续保色

注意：

- 维持期要静，不要继续乱动

### 6.3 恢复 `Recover`

建议时长：

- `0.22s ~ 0.45s`

主层：

- `FullscreenWash` 退场
- `MistLayer` 消散
- `TargetLayer` 恢复

节奏：

- 背景颜色慢慢回来
- 蓝色只先回到玩家与飞剑附近
- 最后世界完全回常态

注意：

- 恢复比进入更慢
- 恢复不是瞬间关掉


## 7. 每层推荐 Godot 组件

### 7.1 全屏状态层

- `ColorRect`
- `ShaderMaterial`

用途：

- 统一屏幕级 wash

### 7.2 纸纹与笔触

- `TextureRect`
- `Sprite2D`

用途：

- 大图贴图层

### 7.3 裂墨与剑光

- `Sprite2D`
- `Line2D`（只在需要真正的方向可塑性时）

用途：

- 裂墨主形
- 剑光 streak

建议：

- 裂墨优先贴图
- 剑光可贴图或 `Line2D`

### 7.4 墨滴与冷雾

- `GPUParticles2D`
- `ParticleProcessMaterial`

用途：

- 入场墨滴
- 持续轻雾
- 细碎浮尘

### 7.5 时间控制

- `AnimationPlayer`
- `Tween`

用途：

- 驱动透明度、缩放、位移、材质参数


## 8. 代码和节点的职责边界

### 8.1 代码输入

代码只向 `TimeStopFx` 提供这些数据：

- `is_active`
- `enter_triggered`
- `sustain_strength`
- `recover_ratio`
- `player_screen_pos`
- `focus_screen_pos`
- `direction`
- `active_targets`

### 8.2 节点内部自己解决的事

节点内部自己解决：

- 哪一层何时显示
- 哪张贴图用在哪
- 哪个粒子什么时候喷
- 哪个参数在进入 / 维持 / 恢复三段怎么变化

### 8.3 禁止继续塞进主渲染器的东西

后续不建议继续放在 `game_renderer.gd` 的内容：

- 裂墨几何
- 大笔触造型
- 目标墨斑贴图
- 墨滴粒子喷发
- 时停主 wash 的最终合成节奏


## 9. 专业制作顺序

### 9.1 先做独立测试场景

不要一开始就挂回主战斗。

先做：

```text
scenes/lookdev/TimeStopFxLookdev.tscn
```

提供三个按钮或快捷键：

- 播放进入
- 播放维持
- 播放恢复

目的：

- 先单独把演出做成立

### 9.2 再回接主战斗

主战斗里只做：

- 开始时调用进入
- 保持时更新强度
- 结束时调用恢复

### 9.3 最后再做调参面板

不要一开始暴露几十个参数。

第一轮只开放：

- wash 强度
- 裂墨长度
- 裂墨宽度
- 墨滴量
- 雾强度
- 目标墨斑强度


## 10. 第一版正式节点化的最小范围

如果要用最低成本进入正式做法，建议第一版只做：

1. `ColorRect + shader` 的全屏 wash
2. `Sprite2D` 的裂墨双层
3. `GPUParticles2D` 的墨滴 burst
4. `Sprite2D` 的目标墨斑池
5. `AnimationPlayer` 的进入 / 恢复轨道

先不要一口气做：

- 复杂背景动态笔触
- 多层粒子循环
- 额外镜头扭曲
- 太多 shader 分支

因为第一版的核心验证问题只有两个：

- 玩家能不能看懂“时停进入”
- 时停维持时读屏是不是清楚


## 11. 当前项目建议的落地路径

### 11.1 当前阶段最合理的方案

建议：

- 保留 `main.gd` 中的玩法时停逻辑
- 新建独立 `TimeStopFx.tscn`
- 用节点方式重做表现
- 旧 `game_renderer.gd` 里的程序化时停逐步退役

### 11.2 与当前项目原则一致的地方

这套方案依然保护了当前项目最重要的优点：

- 飞剑仍然是唯一高亮主体
- 目标和敌人仍然优先可读
- 时停作为状态层成立
- 动作声明和状态氛围被分离

### 11.3 下一步该验证什么

下一轮内部验证只看这几个问题：

- 进入时有没有“沿飞剑方向撕开”的动态感
- 维持期背景是否水墨化但不抢主体
- 飞剑和目标是否比背景亮两个层级
- 恢复期颜色是否平稳回来
- 关掉所有 debug 信息后，画面是否仍然成立


## 12. 一句话结论

Godot 里这类时停效果的专业实现方式应该是：

`代码驱动时机 + 节点分层承载造型 + 粒子提供动态 + shader提供状态底色 + AnimationPlayer负责编排节奏。`

而不是：

`在主渲染器里继续堆公式，直接画出最终特效。`

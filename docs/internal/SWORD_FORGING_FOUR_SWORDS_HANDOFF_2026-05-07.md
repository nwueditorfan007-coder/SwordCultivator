# 四把锻造验证剑实现交接

日期：2026-05-07

本文用于新会话直接接手实现四把锻造验证剑。重点不是完成完整锻造系统，而是在当前可运行战斗中验证“剑的物理结构推导战斗形态”是否成立。

## 1. 必读上下文

先读：

- `docs/project/CURRENT_PROJECT_DESIGN.md`
- `docs/decisions/PROJECT_DECISIONS.md`
- `docs/discussions/SWORD_FORGING_PHYSICAL_ARCHETYPE_DISCUSSION_2026-05-07.md`

如果当前实装与定案文档冲突，实现时以 `docs/project/CURRENT_PROJECT_DESIGN.md` 为准。

当前主玩法事实：

- `左键点按`：近战挥剑，可弹反，可赚剑意。
- `左键长按约 0.10s`：启动并维持剑阵持续输出。
- `右键点按`：御剑点刺。
- `右键长按`：御剑连斩，飞剑跟随鼠标持续切割，松开后回收。
- 御剑飞出期间触发子弹时间。
- 剑阵由鼠标距离连续决定：
  - 近距离：环阵
  - 中距离：扇阵
  - 远距离：贯穿阵
- 当前剑阵有 12 把可见飞剑库存，状态包括 `ready`、`outbound`、`returning`。
- 不增加新战斗按键，不让系统自动替玩家选择环/扇/贯。

## 2. 设计目标

实现四把固定测试剑，用来验证锻造方向：

**玩家锻造的是剑的身体，系统推导出剑的战斗形态。**

第一版不要做完整锻造编辑器，不要做材料掉落，不要做剑灵性格。先做四把可切换的预制剑，让玩家能在同一套输入下感到四种剑确实来自不同物理结构。

核心验证问题：

- 玩家是否能说出“这把剑为什么这样打”？
- 玩家是否能在 `普攻 / 御剑点刺 / 御剑斩击 / 环阵 / 扇阵 / 贯穿阵` 六个动作上都感到剑相差异？
- `轻灵` 与 `重势` 是否改变整套动作的节奏和承诺感，而不只是右键长按？
- `出势` 与 `归锋` 是否改变整套动作的收益时机，而不只是右键回收？
- `凝锋` 与 `分光` 是否改变整套动作的力量组织方式，而不只是贯穿阵或扇阵？

## 3. 锻造模型边界

本次只实现结果，不实现完整锻造流程。

概念层：

- `主材`：决定密度、硬度、韧性、导灵性。
- `剑身`：决定重量、刃压、转动惯量。
- `配重`：决定重心，影响出势或归锋。
- `灵槽`：决定导灵结构，影响凝锋或分光。

战斗层只需要落到三条剑相轴：

| 剑相轴 | 两端 | 战斗含义 |
| --- | --- | --- |
| 剑重 | 轻灵 / 重势 | 右键御剑的操作语法 |
| 剑衡 | 出势 / 归锋 | 伤害重点在出剑还是收剑 |
| 剑路 | 凝锋 / 分光 | 12 剑合成一线还是铺成多线 |

### 3.1 全输入覆盖原则

每条剑相都必须影响六个动作：

- `普攻`
- `御剑点刺`
- `御剑斩击`
- `环阵`
- `扇阵`
- `贯穿阵`

不要把剑相做成“某一个技能的变体”。玩家应该从整套战斗输入里感到这把剑的身体结构不同。

实现时优先做成轴向 modifier：

`最终动作 = 基础动作 + 剑重 modifier + 剑衡 modifier + 剑路 modifier`

不要为四把剑分别硬写四套完全独立技能。四把验证剑只是三条轴组合后的测试样本。

注意：

- `轻灵` 和 `重势` 是同一条轴的两端，不能同时作为高值主属性。
- `出势` 和 `归锋` 是同一条轴的两端。
- `凝锋` 和 `分光` 是同一条轴的两端。
- `锋锐`、`坚韧`、`贯穿度`、`灵导性`等暂不作为主形态，只能以后作为副指标。

## 4. 三条剑相对六个动作的基础影响

### 4.1 剑重：轻灵 / 重势

剑重决定节奏、惯性和承诺感。

| 动作 | 轻灵 | 重势 |
| --- | --- | --- |
| 普攻 | 快速斩，后摇短，适合连续贴身补刀和弹反后追击 | 重斩，后摇更有承诺，但击退、破势、弹反反馈更强 |
| 御剑点刺 | 快速点刺，快速回收，适合切 Silk 和补残血 | 重刺钉点，命中有停顿、打断、震退 |
| 御剑斩击 | 保留当前右键长按滑动追杀，飞剑贴鼠标 | 右键长按改为压剑线，按住蓄势，松开重斩 |
| 环阵 | 高速近身游剑，偏切贴脸敌人 | 近身剑罩/剑墙，偏清弹、抗压、反震 |
| 扇阵 | 高频薄剑幕，连续扫 | 低频厚横扫，推开一侧战线 |
| 贯穿阵 | 逐把细刺，快而连续 | 12 剑合成巨剑劈下或贯下 |

### 4.2 剑衡：出势 / 归锋

剑衡决定收益发生在出剑时，还是回收时。

| 动作 | 出势 | 归锋 |
| --- | --- | --- |
| 普攻 | 向前斩出的第一段更强 | 收剑回身时带二段回刃或回身切割 |
| 御剑点刺 | 飞出去命中最强 | 点刺只是挂线，回收路径更强 |
| 御剑斩击 | 按住期间持续切割更强 | 按住期间偏布线，松开回收爆发 |
| 环阵 | 剑阵向外顶开压力 | 剑阵向内收束，回卷清周身 |
| 扇阵 | 扇形扫出去的首波强 | 扇阵扫出后回卷二段强 |
| 贯穿阵 | 贯出去那一下强 | 穿出后归线斩强 |

### 4.3 剑路：凝锋 / 分光

剑路决定力量是集中成一线，还是分成多线。

| 动作 | 凝锋 | 分光 |
| --- | --- | --- |
| 普攻 | 单条清晰剑弧，窄但深 | 多道浅剑影，宽但单段轻 |
| 御剑点刺 | 单点高质量刺击 | 主刺旁带短延迟分光 |
| 御剑斩击 | 一条干净切线，适合精准切 Silk | 鼠标轨迹有延迟复刻剑光 |
| 环阵 | 紧密单环，像一条切割带 | 内外双层薄环，不出现缺口 |
| 扇阵 | 扇面更窄、更密、更狠 | 扇面更宽，多线覆盖 |
| 贯穿阵 | 单线贯穿或巨剑贯穿 | 两到三条并行贯穿线 |

## 5. 四把验证剑

### 5.1 风砂银分光剑

剑相：

- 剑重：轻灵
- 剑衡：出势
- 剑路：分光

物理解释：

- 低密度，薄刃，导灵性好。
- 灵槽偏分流，剑气容易拆成多线。
- 重心不重，适合连续追杀。

战斗表现：

- `普攻`：快速多影斩，向前第一段更强，适合贴身补刀。
- `御剑点刺`：快速主刺并带短延迟分光，出剑命中收益高，回收普通。
- `御剑斩击`：保留当前滑动连斩，主剑贴鼠标，轨迹带延迟复刻剑光。
- `环阵`：高速内外双层薄环，主要清近身压力，不做长射程输出。
- `扇阵`：高频多线剑幕，覆盖更宽，单线伤害较轻。
- `贯穿阵`：两到三条并行细线贯穿，适合清多目标线。

玩家感受：

**这把剑很轻，适合追着切，并把剑光铺开。**

### 5.2 寒玉归锋剑

剑相：

- 剑重：轻灵
- 剑衡：归锋
- 剑路：凝锋

物理解释：

- 硬、冷、窄刃，导灵纯净。
- 剑形细而集中，灵力不分散。
- 配重和导灵回路让收剑路径更干净。

战斗表现：

- `普攻`：轻快单线斩，正向伤害一般，收剑回身有清晰回刃。
- `御剑点刺`：单点高质量刺击，点刺可切 Silk 和弱点，但真正爆点在回收凝线。
- `御剑斩击`：仍是滑动连斩，但按住期间偏布线，松开时回收路径形成高质量凝线斩。
- `环阵`：紧密单环，外放后向内收束，偏近身切割和回卷清周身。
- `扇阵`：扇面较窄，扫出后回卷形成二段切线。
- `贯穿阵`：逐把细刺或单线贯出后，回收时形成一条强归线。

玩家感受：

**这把剑不是一直切最强，而是收回来的那一下最讲究。**

### 5.3 玄铁巨阙剑

剑相：

- 剑重：重势
- 剑衡：出势
- 剑路：凝锋

物理解释：

- 高密度，厚剑脊，重心靠前。
- 刚性高，适合多剑合一。
- 导灵结构集中，不适合花哨分光。

战斗表现：

- `普攻`：厚重单线斩，后摇更有承诺，但击退、破势、弹反反馈更强。
- `御剑点刺`：重刺钉点，出剑命中是主要爆点，回收只是收尾。
- `御剑斩击`：从当前滑动连斩改为压剑线。按住右键进入蓄势线，鼠标方向决定剑势线，松开后重剑沿线轰出。
- `环阵`：12 剑形成近身剑罩/剑墙，强化清弹、抗近身压力和反震。
- `扇阵`：低频厚重单线横扫，击退一侧敌群。
- `贯穿阵`：12 剑合成巨剑劈下或贯下，适合 Boss 窗口与重甲目标。

玩家感受：

**这把剑不是追杀，而是提前压住一条战线。**

### 5.4 星陨回渊剑

剑相：

- 剑重：重势
- 剑衡：归锋
- 剑路：分光

物理解释：

- 密度高，灵容量高。
- 配重和导灵回路形成强回流。
- 灵槽带分流，出剑后能留下多道回卷轨迹。

战斗表现：

- `普攻`：重斩带多道浅剑影，正向压迫强，回身时多线回刃。
- `御剑点刺`：重刺钉点后，回收路径带明显多线二段。
- `御剑斩击`：仍采用重势压剑线，但松开后的出势不是唯一重点，随后回收会拖出多道裂线。
- `环阵`：近身剑罩外扩后回收，形成内外层回卷清场。
- `扇阵`：厚重扇扫后，多道剑光回卷。
- `贯穿阵`：巨剑贯出后分裂回流，回收路径造成二段或多段收束伤害。

玩家感受：

**这把剑出手重，但真正可怕的是回来的回渊。**

## 6. 实现建议

第一版建议做成调试用预制剑，而不是完整装备系统。

建议新增或集中配置一个剑相 profile：

- `sword_id`
- `display_name`
- `weight_axis`: `light` 或 `heavy`
- `balance_axis`: `outward` 或 `returning`
- `path_axis`: `focused` 或 `split`
- `material_tags`
- `melee_modifiers`
- `point_strike_modifiers`
- `sword_slice_modifiers`
- `ring_array_modifiers`
- `fan_array_modifiers`
- `pierce_array_modifiers`
- VFX 色彩/尾迹参数

可能需要关注的文件：

- `scripts/system/main.gd`
- `scripts/system/sword_array_config.gd`
- `scripts/system/sword_array_controller.gd`
- `scripts/combat/attack_profiles.gd`
- `scripts/vfx/sword_vfx_profile.gd`
- `resources/vfx/sword_vfx_profile_default.tres`

实现顺序建议：

1. 先加 profile 数据结构和调试切换方式。可以临时使用 F 键或调试菜单，但不要纳入正式战斗按键设计。
2. 先实现 `轻灵` 与 `重势` 对六个动作的差异，尤其是御剑斩击和环阵。
3. 再实现 `出势` 与 `归锋` 对六个动作的收益时机差异，尤其是普攻回刃和御剑回收。
4. 最后实现 `凝锋` 与 `分光` 对六个动作的形态差异，尤其是扇阵和贯穿阵。
5. 再组合出四把验证剑。
6. 每一步都保留默认剑，方便 A/B 对比。

## 7. 最小可验证版本

如果时间有限，先只做两把：

- `风砂银分光剑`
- `玄铁巨阙剑`

因为它们分别代表最清晰的两个极端：

- 轻灵滑动追杀
- 重势压线破阵

只有当这两个手感成立，再做 `寒玉归锋剑` 和 `星陨回渊剑`。否则归锋/分光细节会掩盖核心问题。

## 8. 验收标准

实现后用同一波敌人和 Boss/Silk 机制测试。

必须验证：

- 玩家不看数值，仅凭操作就能分辨轻灵和重势。
- 玩家能在 `普攻 / 御剑点刺 / 御剑斩击 / 环阵 / 扇阵 / 贯穿阵` 六个动作上都感到剑相差异。
- 重势右键不会被误解为“输入延迟”或“剑不听话”，而是“压线蓄势”。
- 归锋剑能让玩家主动考虑松开右键的时机和回收路径。
- 分光剑增加覆盖，但不让 12 把剑画面变脏。
- 贯穿阵仍由鼠标距离触发，不被系统自动替玩家选择。
- 环阵仍是最近距离形态，不能因为某把剑变成长射程输出。
- Silk 处理仍然清楚，不能因为分光/巨剑而看不出切丝责任。
- 普攻差异不能破坏左键弹反作为当前近身生存和剑意建立手段的职责。

## 9. 明确不要做

本轮不要做：

- 完整锻造编辑器。
- 材料掉落和合成流程。
- 剑灵性格对战斗的主动介入。
- 自动索敌替玩家选阵型。
- 新增正式战斗按键。
- 让轻灵与重势同时堆高。
- 大量纯数值词条。
- 只改某一个动作就宣称剑相成立。
- 为每把剑硬写完全独立输入规则，导致后续无法扩展。

本轮要做的是战斗手感验证，不是系统规模扩张。

## 10. 剑灵性格接口预留

剑灵性格暂不实现，但代码结构最好预留事件接口：

- 右键点按命中
- 右键长按持续命中
- 松开右键回收
- 环阵发射/维持
- 扇阵发射/回收
- 贯穿阵命中
- 切断 Silk
- 清掉子弹
- 剑意低
- 玩家低血

后续原则：

**锻造决定剑怎么动，剑灵决定它在这些动作里偏向什么行为。**

不要让剑灵重写锻造形态，否则系统边界会混乱。

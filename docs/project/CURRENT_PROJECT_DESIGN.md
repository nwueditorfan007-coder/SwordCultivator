# 剑修 Demo 当前实装玩法

本文件只描述 `res://scenes/main/Main.tscn` 这条主玩法当前已经实装、可以直接运行验证的内容。

它和 `docs/decisions/PROJECT_DECISIONS.md` 的职责不同：

- 本文件回答“现在代码里实际能玩到什么”
- 定案文档回答“项目后续准备往什么方向收敛”

如果两者冲突，以本文件作为当前实机判断依据。

## 1. 项目定位

- 类型：单屏俯视角动作弹幕原型
- 当前核心识别点：
  - 近战挥剑与弹反
  - 右键御剑带来的子弹时间
  - 鼠标距离驱动的连续变形剑阵
  - `Boss / Puppet / Silk` 机制处理
- 当前不是：
  - “冻结弹幕 -> 主动吸收 -> 再发射”的旧循环
  - “Space 消耗剑丸开启强化剑技”的新定案版本

## 2. 当前实际核心循环

普通战斗里的真实循环更接近：

`走位 -> 左键近战弹反与赚剑意 -> 右键御剑打高价值目标 / 切丝 -> 左键长按用剑阵持续输出 -> 清波次 -> 进入更高压力`

Boss 战里的真实循环更接近：

`处理弹幕与 Puppet 压力 -> 用御剑切断 Silk -> 打开 Boss 受击窗口 -> 用近战 / 剑阵兑现伤害`

因此，当前版本已经不是最早那版“吸收弹幕后再发射”的玩法，也还没有进入“强化剑技爆发窗口主导”的版本。  
它现在更像：

`近战建节奏 + 御剑切机制 + 剑阵常规持续输出 + Boss 开窗`

## 3. 基础操作

- `WASD`：移动
- `左键点按`：近战挥剑
- `左键长按约 0.10s`：启动并维持剑阵持续输出
- `右键点按`：御剑点刺
- `右键长按`：御剑连斩；松开后回收
- `F7`：战斗调试
- `F6`：阵图校准调试

当前主场景中，`Space` 没有绑定战斗功能。项目设置里仍保留了旧输入映射，但主玩法脚本没有使用它。

第一章当前接入了逐步解锁：

- 第 `1` 波：近战
- 第 `2` 波：御剑
- 第 `3` 波：环阵
- 第 `4` 波：扇阵
- 第 `5` 波：贯穿阵

前 `1-5` 波会用更厚的分段刷怪作为单技能练习波，让玩家大约完成 `9-10` 次当前新动作后，再进入下一项能力。

解锁前，飞剑本体仍可作为题材视觉存在，但对应输入会被拦截并给出即时反馈。

## 4. 玩家系统

### 4.1 生命与剑意

- 玩家生命上限为 `100`
- 剑意上限为 `100`
- 当前实装里，剑意主要用于维持左键长按的剑阵持续输出
- 当前实装里，御剑本身不消耗剑意

### 4.2 近战

- 飞剑在玩家身边环绕时，角色处于近战态
- 左键挥剑会打出前方大弧形斩击
- 近战可命中普通敌人与开窗期 Boss
- 近战可弹反敌方子弹，反弹后的子弹会高速飞回去伤敌
- 近战命中、弹反、击杀都会给玩家回剑意
- 飞剑环绕期间，玩家还会持续获得少量自然回剑意

### 4.3 御剑

- 右键点按触发点刺，飞剑直刺目标点后回收
- 右键长按触发连斩，飞剑跟随鼠标持续切割，松开后回收
- 御剑飞出期间会触发子弹时间：
  - 敌方弹幕从 `0.1x` 逐步恢复到 `1.0x`
  - 玩家自身从 `0.85x` 逐步恢复到 `1.0x`
- 御剑可以稳定切 `Silk`
- 御剑不能把 `Puppet` 当普通敌人直接砍死

### 4.4 剑阵

- 当前剑阵不是“先吸收弹幕再发射”，而是玩家自带一组可见飞剑库存
- 当前常驻库存是 `12` 把飞剑
- 这些飞剑始终以实体形式存在，并在三种状态之间周转：
  - `ready`：挂在玩家周围，可立即发射
  - `outbound`：已射出，正在攻击
  - `returning`：攻击结束后回收
- 飞剑出入阵轨迹当前已进入新一轮抛光期：
  - 当前代码里已有轻量流动化原型
  - 但终版实现依据以 `docs/systems/SWORD_ARRAY_FLOW_TRAJECTORY_DESIGN.md` 为准
- 阵型由鼠标距离连续决定：
  - 近距离：`环阵`
  - 中距离：`扇阵`
  - 远距离：`贯穿阵`
- 视觉形变和发射判定已经分层：
  - 视觉层跟随原始鼠标距离
  - 发射层使用平滑后的控制距离，降低误切阵型

当前三形态的实际职责如下：

- `环阵`：近身护体与解压
  - 一次尽量整批射出当前可用飞剑
  - 持续维持时可清掉玩家身边和环阵附近的近身子弹
- `扇阵`：中距离面扫
  - 每轮发射半批飞剑
  - 同批飞剑完成本轮扫射后再一起回收
- `贯穿阵`：远距离破线
  - 逐把射出
  - 每把飞剑可穿多个目标

要注意的是：

- 当前剑阵属于日常可用的常规输出位
- 它会持续消耗剑意
- 它不是只在“强化窗口”里才出现的爆发层

## 5. 敌人与波次

### 5.1 普通敌人

- `Shooter`：保持中距离并发射基础小弹
- `Tank`：缓慢逼近，靠碰撞持续施压
- `Caster`：随机位移并释放环形弹幕
- `Heavy`：发射更慢更重的大弹
- `RingLeech`：高速贴近并围绕玩家形成近身压力
- `DrapePriest`：用连线支援其他敌人，预教学 `Silk` 机制
- `MirrorNeedler`：远距离蓄势发射重针，是高价值处理目标

### 5.2 波次推进

- 默认从第 `1` 波开始
- 第一章前 `9` 波使用手工教学队列与典型构图，设计依据见 `docs/levels/CHAPTER_01_ONBOARDING_WAVE_PLAN.md`
- 这些教学波支持相对玩家位置的指定出生点和分段刷怪延迟，用于形成更清晰的战斗命题
- 教学波数量密度高于最初版本，避免玩家只做少量操作就进入下一课
- 第 `10` 波进入第一章 Boss 考试
- Boss 击败后推进到下一波，不再重复同一波普通刷怪
- 第 `10` 波之后回到通用随机波次，普通波次敌人数按 `3 + wave * 2` 增长
- 后续每 `10` 波进入一次 Boss 战

## 6. Boss / Puppet / Silk

### 6.1 Boss 当前状态

Boss 当前已经实装以下状态：

- `Thousand Silks`
- `Puppet Ambush`
- `Silk Cage`
- `Needle Return`

Boss 有阶段切换：

- 血量低于 `70%` 进入第二阶段
- 血量低于 `30%` 进入第三阶段

### 6.2 Puppet / Silk 规则

- `Puppet` 是 Boss 机制体，不是普通敌人
- 当前实现里，`Puppet` 不接受近战、御剑、剑阵的常规伤害处理
- `Silk` 是真正要切断的目标
- 当对应 `Silk` 被切断后，关联的 `Puppet` 会立刻失效/死亡

### 6.3 Boss 受击窗口

- Boss 紫色本体表示闭窗，不接受常规伤害
- Boss 黄色本体表示破绽窗口，可以被近战、御剑、剑阵兑现伤害
- `Puppet Ambush` 期间清空有效 `Silk` 后，会直接打开短暂破绽

## 7. 当前实现与文档的关键差异

下面这些点最容易让人被旧文档误导：

- 已移除：
  - `冻结弹幕`
  - `主动吸收`
  - `吸收后围绕玩家的弹丸库存`
- 还未实装：
  - `剑丸资源`
  - `Space 开启强化剑技`
  - `常规态不发射、仅强化态爆发剑阵`
- 当前实际存在：
  - `12` 把常驻可见飞剑库存
  - 左键长按直接进入常规剑阵持续输出
  - 御剑不消耗剑意

## 8. 对核心玩法的当前判断

如果只看“项目最初想验证的战斗骨架”，当前实现并没有跑偏，反而更明确了：

- 近战仍然负责贴身生存、弹反和资源建立
- 御剑仍然是最有辨识度的机制工具
- 剑阵仍然是项目的第二层战斗语言，并且连续变形手感已经成为签名体验
- Boss 仍然围绕 `切丝开窗` 展开

真正的偏离，不在“题材和战斗骨架”层，而在“资源职责与循环排序”层：

- 当前实现还没有跟上后续定案的 `剑丸爆发窗口` 方案
- 当前剑阵仍然在和御剑争夺日常主动输出位
- 当前 `Space / 强化剑技` 这一层缺席，导致循环仍停在“常规剑阵版”

因此，当前结论应理解为：

**项目没有偏离最早那层核心玩法，但它确实停留在一个介于“旧吸收版”和“新强化版”之间的中间实现阶段。**

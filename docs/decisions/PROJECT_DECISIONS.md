# 项目已定案设计决策

本文件只记录当前已经确认、可以作为实现依据的设计结论。

注意：

- 本文件描述的是项目准备收敛到的目标方向
- 当前主场景里已经实际落地的玩法，请优先对照 `docs/project/CURRENT_PROJECT_DESIGN.md`

如果某些条目与当前代码不一致，应先理解为“实现尚未完全跟上定案”，而不是默认代码或文档其中一方已经失效。

## 1. 战斗调优方法

- 战斗讨论默认以玩家实机感受为起点，而不是先从抽象数值出发。
- 每次调优都先判断玩家感受是否正确，再定位系统根因，再决定最小可验证原型。
- 优先保护已经验证有效的爽感，不为了解决一个问题顺手牺牲项目签名体验。

## 2. 当前核心战斗方向

- 核心战斗由 `近战`、`御剑`、`剑阵` 和 `Boss 机制处理` 共同构成。
- `近战`负责贴身兑现、弹反、近身压制与剑意获取。
- `御剑`负责切机制、制造窗口、提供高价值中远程处理。
- `剑阵`保留为持续可见的飞剑形态层，并在强化窗口内根据形态承担近中远三类持续输出职责。
- `强化剑技`是主动触发的短时爆发窗口，不是日常常驻态。

## 3. Boss / Puppet / Silk 机制

- Puppet 是 Boss 机制体，不是普通敌人。
- Puppet 不显示普通生命条。
- Puppet 不能被飞剑、近战或剑阵当作普通敌人直接击杀。
- Puppet 的主要死亡条件是对应 Silk 被切断。
- 在 `Puppet Ambush` 阶段，清空有效丝线后 Boss 进入脆弱状态。

## 4. 剑阵输入与控制原则

- 不增加新按键。
- 不使用滚轮切换阵型。
- 不采用自动战场判断来代替玩家选择阵型。
- 阵型主导语义仍由“准星与玩家距离”决定。
- 必须保留当前连续变形的视觉手感，不做整段硬锁阵型。
- 不用“更高纯数值收益”直接补偿操作税，优先修控制结构。

## 5. 剑阵视觉层与发射层分离

- `视觉变形`继续使用原始鼠标距离驱动，保持连续、丝滑、可读。
- `发射控制`使用独立的控制距离与控制阈值，降低持续射击中的误切形态。
- 因此，“操作距离”和“战术射程”被明确拆开，不再硬绑定。

## 6. 剑阵三形态职责分工

- `环阵`：近距离护身、解压、清周身威胁。
- `扇阵`：中距离面扫、稳定压制一侧战线。
- `贯穿阵`：远距离破线、处理高价值目标。

这三个职责优先通过：

- 射程层级
- 覆盖方式
- 周转语义
- 稳定性

来区分，而不是只靠面板伤害区分。

## 7. 强化剑技资源规则

- `Space` 消耗 `1` 个剑丸资源，开启 `5s` 强化剑技。
- 长按 `左键` 只在强化窗口内发射攻击型剑阵。
- 常规状态下，`剑阵`主要承担视觉形态、飞剑库存可读性与爆发前姿态准备。
- 强化期间，`剑阵`不消耗剑意。
- `御剑`仍然继续消耗剑意。
- 强化开启时，会立刻补齐并归位剑阵飞剑，保证爆发窗口可立即兑现。
- 如果玩家正在按住左键，强化开启时应立即接入爆发剑阵，而不是重新要求一次常规蓄阵。

## 8. 日常循环与爆发循环分工

- 日常循环中，主动消耗型输出位优先留给 `御剑`。
- `近战`负责建立节奏、处理贴身压力、执行弹反。
- `剑阵`不再与 `御剑`争夺同一个常规剑意消耗位。
- `剑阵`的三形态选择与读感保留，但主要在强化窗口内兑现。
- 这样做的目标是让玩家形成更清晰的肌肉记忆：
  - 平时：`左键近战`、`右键御剑`
  - 爆发：`Space 消耗剑丸开启强化剑技`

## 9. 剑阵可读性与战场信息量

- 剑阵飞剑必须作为可见实体存在，不退回为纯数字库存。
- 场上允许出现可见缺口，用于传达当前可用飞剑数量。
- 飞剑返航连线不作为正式战斗表现保留。
- 阵型预览不常驻，只在蓄阵时显示。
- UI 和场上图形的首要目标是服务战斗读感，而不是解释系统。

## 10. 当前已定案的飞剑行为原则

- 飞剑不能“碰一下就立刻折返”。
- 飞剑射出后至少完成最小出击距离，再允许返航。
- 命中后仍应保留短掠进或后续前飞，而不是立即失去出击价值。
- 不同阵型可以在未来拥有不同的回收语义，但当前统一原则是：先完成一次有效出击，再回收。

## 11. 当前已定案的射程层级

- 环阵必须是三种形态里最近的。
- 扇阵处于明确中程。
- 贯穿阵保持最远。
- 环阵即使更容易维持，也不能因此获得过长的实际攻击距离。

## 12. 当前已验证的环阵调优方向

- 环阵的问题优先通过“更近、更稳、更能清周身威胁”解决，而不是先加远程能力。
- 环阵的可靠性优先来自更近的出击目标、更短射程、更强近身命中宽度与贴脸威胁处理能力。
- 强化环阵是当前环阵身份的主要兑现层。
- 爆发态环阵允许拥有克制的近身清弹能力，用来补足爆发期间左键被剑阵占用后产生的防御空档。
- 常规态环阵不提供清弹能力，常规态防弹职责仍归普通攻击弹反。
- 常规态不再承担攻击型环阵输出职责，因此后续环阵优化重点转向强化窗口内的近程护身表达。

## 13. 技能不可释放反馈原则

- 当 `御剑`、`剑阵` 或 `强化剑技` 因资源或库存条件不足而无法释放时，必须提供即时可读的反馈。
- 不同失败原因应尽量可区分，至少要区分：
  - `剑意不足`
  - `剑阵飞剑库存不足 / 正在回收`
  - `剑丸不足`
- 这类反馈不应完全依赖玩家自己盯数值条推断，最好同时具备明确的 UI、状态提示或其他即时表现。

当前实现已经采用：

- 状态文字提示
- 对应 HUD 行的短促闪烁染色
- 失败提示节流，避免长按时刷屏

后续如仍然不够直观，再考虑补失败音效或更强的局部 UI 动效。

## 14. 剑丸经济节奏原则

- `剑丸`的产出要支持“计划一次强化窗口”，但不能高到让强化几乎成为常驻能力。
- 当强化手感已经成立、但资源过剩时，优先收紧 `剑丸掉落频率`，而不是直接削弱强化本身的爽感。
- `剑丸`应被玩家理解为爆发许可，而不是稳定溢出的第二蓝条。

## 15. 普通攻击弹反手感原则

- 普通攻击弹反出去的子弹必须足够快，读感应接近“斩开后立刻反杀”。
- 如果弹反看起来拖沓，优先提高反弹子弹速度，先修观感和兑现速度，再考虑其他数值。

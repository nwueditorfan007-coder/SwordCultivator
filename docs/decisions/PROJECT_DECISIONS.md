# 项目已定案设计决策

本文件只记录当前已经确认、应作为实现依据的设计结论。

## 1. 核心战斗方向

- 项目核心不是普通平砍清怪。
- 核心循环是：
  - 近战积累资源
  - 冻结弹幕
  - 吸收冻结弹幕
  - 御剑切机制
  - 在窗口期完成高价值输出

## 2. 御剑伤害逻辑

- 飞剑只有在“穿过目标”时才能造成伤害。
- 飞剑停留在敌人或 Boss 身上不会持续造成伤害。

## 3. 御剑时停逻辑

- 御剑启动后触发时停衰减曲线。
- 当前定案参数：
  - 初始时间倍率：`0.1x`
  - 恢复时长：`2.0s`

## 4. Puppet / Silk 机制

- Puppet 是 Boss 机制体，不是普通敌人。
- Puppet 不显示普通生命条。
- Puppet 不能被飞剑、近战、发射弹丸、必杀直接击杀。
- Puppet 的主要死亡条件是对应 Silk 被切断。
- 在 `Puppet Ambush` 阶段，清空有效丝线后 Boss 进入脆弱状态。

## 5. Boss 受击逻辑

- Boss 当前允许被近战攻击命中。
- Boss 在脆弱阶段或指定可受击阶段可承受输出。

## 6. 剑阵系统控制原则

- 不增加新按键。
- 不使用滚轮切换。
- 不采用自动战场判断。
- 剑阵形态由“准星与玩家距离”决定。

## 7. 剑阵系统 V1 阈值

- `0 - 160`：环阵
- `160 - 420`：扇阵
- `420+`：贯穿阵

## 8. 剑阵系统 V1 角色分工

- 环阵：近身护体 / 解围
- 扇阵：中距离主力 / 精准清场
- 贯穿阵：远程破线 / 打 Boss / 打高价值目标

## 9. 剑阵系统 V1 当前边界

- V1 先做三段形态和实时预览。
- V1 不做：
  - 融合升阶
  - 自动追踪
  - 连续无级形变
  - 手动模式切换

# 剑阵手感迭代纪要

- 日期：`2026-04-18`
- 性质：讨论与迭代纪要
- 状态：部分内容已验证并落地，部分内容作为下一阶段优化储备
- 主题：强化剑技可靠性、剑阵形态职责、连续变形控制、环阵近程化、后续回收逻辑差异化

## 1. 这份文档记录什么

这份文档用于收拢最近几轮围绕剑阵手感的设计讨论，明确区分三类内容：

- 已经通过实现与实机验证的方向
- 当前已形成共识，但还未实现的优化储备
- 下一轮继续验证时应该重点观察的问题

它不是最终定稿文档，但会作为后续实现排序与讨论恢复上下文的重要依据。

## 2. 最近几轮讨论形成的总原则

### 2.1 以玩家实机感受为起点

本轮迭代不是先问“哪个参数更合理”，而是先问：

- 玩家哪里觉得卡手
- 哪个按钮不再值得信任
- 哪个画面虽然热闹但不好看
- 哪个形态的职责没有被兑现

随后再把这些体验翻译成系统问题。

### 2.2 优先保护连续变形的帅感

已经明确形成共识：

- 连续变形是当前剑阵系统最有辨识度的体验之一
- 不应为了提升阵型稳定性，直接用“发射期间整段锁阵型”破坏这种体验

因此后续所有控制优化，都以“不破坏连续变形视觉手感”为前提。

### 2.3 不用纯数值去补操作税

已经明确形成共识：

- 环阵和扇阵相对更难稳定维持，这是真实存在的输入负担
- 但这类问题不应优先用“更高伤害”去补偿
- 更健康的做法是先修控制结构，再看是否还需要补形态收益

## 3. 已经落地并验证过的改动

## 3.1 强化剑技从“卡手按钮”改为“可靠爆发窗口”

已经实现：

- `Space` 消耗 `1` 个剑丸资源，开启 `5s` 强化剑技
- 强化期间，剑阵不再消耗剑意
- 强化开启时，飞剑会立即补齐并归阵
- 如果玩家此时已在持续蓄阵，强化可以直接接上第一轮发射

这一版的核心目标是把强化从“还要再检查一次资源是否足够”改成“之前的准备终于兑现了”。

最新反馈结论：

- 强化环阵的爽感已经明显成立
- 强化按钮的可信度已大幅提升

## 3.2 飞剑不再“碰一下就回头”

已经实现：

- 飞剑拥有最小出击距离
- 命中后会继续前飞一小段再返航
- 出界或达到最大距离后才回收

这一步的目标是避免大量短促 V 字折返轨迹，让出击更像一次有效掠杀，而不是库存周转动画。

## 3.3 战场信息量做减法

已经实现：

- 移除飞剑返航连线
- 阵型预览不再常驻，只在蓄阵时显示

这一步的目标是减少“解释系统”的画面元素，让玩家更多关注飞剑本体、敌人和弹幕。

## 3.4 视觉变形与发射控制分层

已经实现并确认方向正确：

- 视觉层继续按原始鼠标距离连续变形
- 发射层改为使用独立的 `control_distance`
- `control_distance` 不是瞬时切换，而是每帧平滑追随原始距离

当前平滑速度为：

- 空闲：`12.0`
- 左键按住但未持续发射：`9.0`
- 持续发射中：`6.5`

这一步不是“锁阵型”，而是给发射判定加入惯性，减少手部微抖带来的频繁误切。

## 3.5 发射控制阈值放宽，但视觉阈值不动

视觉阈值仍是：

- `ring_stable_end = 85`
- `ring_to_fan_end = 145`
- `fan_stable_end = 295`
- `fan_to_pierce_end = 400`

当前发射控制层阈值调整为：

- `ring_stable_end = 115`
- `ring_to_fan_end = 179`
- `fan_stable_end = 331`
- `fan_to_pierce_end = 400`

结论：

- 玩家看到的连续变形仍旧丝滑
- 实际发射更容易稳在当前想维持的形态上

## 3.6 三形态的战术射程被明确拉开

已实现的方向：

- 环阵：近程
- 扇阵：中程
- 贯穿阵：远程

这一步和“控制距离”的优化是分开的。  
即使环阵更容易维持，也不允许它因为好控而顺手打到过远位置。

## 3.7 环阵整体近程化与常规态补强

最近一轮已将“所有环阵”一起拉回近身职责，而不是只调强化态。

当前环阵关键参数为：

- `fire_interval = 0.30`
- `sortie_target_offset = 124.0`
- `sortie_max_distance = 190.0`
- `sortie_guidance_max_distance = 88.0`
- `sortie_min_distance = 80.0`
- `sortie_hit_follow_through_distance = 44.0`
- `sortie_hit_radius_bonus = 22.0`
- `sortie_bullet_clear_radius_bonus = 28.0`

这组改动的设计目的不是单纯削短，而是同时做到：

- 环阵飞不远
- 环阵更容易扫掉周身威胁
- 常规环阵不再显得只有强化态才“像技能”

最新反馈结论：

- 当前环阵整体手感已经达到“可以接受并可继续以此为基础推进”的状态
- 但常规环阵仍然还没有完全建立“能稳定清理近身威胁”的效果，这一项仍需继续优化

## 4. 当前已确认的形态职责理解

### 4.1 环阵

- 职责是近距离护身、解压、清周身威胁
- 优势不在远程覆盖，而在近身可靠性
- 如果后续继续加强，也优先加强“近身兜底能力”，而不是加远程能力

### 4.2 扇阵

- 职责是中距离面扫与侧向压制
- 它的价值在于稳定地处理一侧战线，而不是像环阵一样全向护身

### 4.3 贯穿阵

- 职责是远距离破线与高价值目标处理
- 它应该拥有最清晰的直线压制感和最长的实际攻击距离

## 5. 已经形成方案、但尚未完全做完的优化储备

## 5.1 三形态回收逻辑差异化

这是已经明确提出的下一阶段方向，目前进度如下：

- 环阵：更像一轮掠杀，命中后继续掠过去再回
- 扇阵：整批回收已进入当前实现，但仍需要继续调手感
- 贯穿阵：稳定穿透已进入当前实现，但“整套回收语义差异化”仍未全部完成

### 5.1.1 为什么要做这件事

虽然当前统一版“先完成出击再回收”已经比早期版本好很多，但三种阵型仍然还不够“像自己”。

当前不足在于：

- 环阵还没有完全打出掠杀感
- 扇阵虽然已进入整批回收方向，但是否已经足够像“一次成片扫面”仍需继续验证
- 贯穿阵虽然已经进入稳定穿透方向，但还没有完成整套身份抛光

### 5.1.2 推荐实现顺序

推荐按以下顺序推进：

1. `贯穿阵` 先做稳定穿透  
   当前已实现基础版，核心是 `penetration count + per-target hit cooldown`

2. `环阵` 再做更强的掠杀后返航  
   当前已经有命中后前飞一段的基础，可在此基础上继续加深身份

3. `扇阵` 最后做整批回收  
   当前已完成基础版，核心是 `batch_id / 批次状态`

### 5.1.3 当前对体量的判断

- `环阵 + 贯穿阵` 原型：中小到中等体量
- `再加扇阵整批回收`：中等体量，结构成本明显上升
- 真正做漂亮的成本主要在回收批次逻辑与手感调参

### 5.1.4 当前已落地的贯穿阵版本

当前已经先完成了推荐顺序里的第一步：

- 贯穿阵飞剑不再以“首个命中目标”作为返航触发点
- 每把贯穿飞剑拥有独立的 `penetration_targets`
- 同一把飞剑对同一目标存在短暂 `rehit cooldown`

这一版的目标是：

- 让贯穿阵更稳定地连续穿过多个目标
- 避免在同一个敌人或 Boss 身上抖动反复判定
- 保持结构轻量，不提前引入扇阵批次回收所需的更重状态层

### 5.1.5 当前已落地的扇阵版本

当前已经先完成了扇阵的第一版整批回收：

- 同一轮扇阵发射出来的飞剑共享 `batch_id`
- 单把飞剑完成本轮扫射后，会等待同批其他飞剑完成
- 当整批成员都完成本轮扫射后，再一起转入返航

这一版的目标是先把“整批扫出、整批回收”的读感建立起来。

后续仍需继续验证的是：

- 它是否已经足够像一次成片扫面，而不只是“几把剑碰巧一起回头”
- 等待同批成员完成时，停顿感是否自然
- 是否还需要补额外的批次表现或节奏强化

## 5.2 环阵后续只做近身增强，不做远程扩权

已经形成明确共识：

- 环阵即使后续仍需优化，也不该再向远程扩张
- 如果后续还觉得环阵偏弱，应优先检查：
  - 是否近身覆盖还不够连续
  - 是否清弹/清贴脸威胁还不够稳
  - 是否与扇阵的职责仍有重叠

## 5.3 当前新增待解决问题

### 5.3.1 常规环阵仍然不够像“近身清威胁工具”

当前最新实机反馈表明：

- 强化环阵的手感已经成立
- 但常规环阵仍然没有充分打出“清理近身威胁”的效果

这说明当前问题已经不再主要是“环阵太远”，而更偏向：

- 常规态的近身覆盖仍不够连续
- 常规态的威胁清除效率仍然不够可靠
- 玩家在贴脸压力下，仍不能足够信任常规环阵会替自己清出安全区

后续继续优化时，应优先补：

- 近身可靠性
- 连续覆盖感
- 清弹 / 清贴脸目标的兑现率

而不是重新给常规环阵加远程能力。

### 5.3.2 资源不足反馈已进入当前实现，但还需要继续验证强度

这一项已经进入当前代码实现：

- `剑意不足`
- `飞剑未回收 / 库存不足`
- `剑丸不足`

现在都会触发明确的状态文字提示。

同时还补了两层轻量 UI 反馈：

- 能量相关失败会短促染色 `剑意 / 剑丸` HUD 行
- 飞剑未回收会短促染色 `得分 | 飞剑 x / y` HUD 行

为了避免按住输入时刷屏，当前实现还加了轻度提示节流。

这一版解决的是“按钮不可信”的核心问题，但还需要继续验证：

- 现在的提示是不是已经足够直观
- 是否还需要补一个更明确的失败音效
- 当前 HUD 染色强度是否合适，还是还略显保守

### 5.3.3 剑丸掉落频率已收紧，进入经济节奏验证阶段

最新实机反馈表明：

- 当前剑阵整体爽感已经成立
- 但 `剑丸` 掉落频率偏高，开始稀释强化作为爆发窗口的身份

因此这轮没有削强化本体，而是先收紧资源产出：

- 精英掉落概率下调
- 精英保底间隔拉长
- Boss 周期掉落间隔拉长
- Boss 单次掉落概率下调

这版的设计目标是：

- 保住强化剑技已经成立的爽感
- 让 `剑丸` 回到“值得期待的爆发资源”
- 避免强化几乎顺手常驻

下一轮实机要重点观察的是：

- 现在是否仍然能稳定攒出强化窗口
- 剑丸节奏是“更健康”，还是开始有点太抠
- 强化是否重新回到“想什么时候开就什么时候开”的稀释状态之外

### 5.3.4 普通攻击弹反速度已提升，进入观感验证阶段

这一轮还同步提高了普通攻击弹反出去的子弹速度。

这样做的原因不是单纯为了加伤，而是为了修正读感：

- 之前的弹反更像“慢慢推出去”
- 它不够像一次果断、干脆的反杀动作

这版的目标是让弹反更接近：

- 挥开来弹
- 立刻回击
- 更能建立近战的掌控感

下一轮实机要重点观察的是：

- 现在的弹反是否终于像“斩开后立刻反杀”
- 速度是否刚好，还是开始过快到影响轨迹读感
- 弹反命中敌人时是否比以前更有兑现感

### 5.3.5 常规攻击型剑阵已从日常循环中移除

在继续验证后，当前已经形成新的结构判断：

- `常规剑阵` 和 `御剑` 之前共用 `剑意`
- 它们同时在争夺日常主动输出位
- 即使把常规剑阵手感修顺，这种结构竞争仍然会让玩家难以下意识判断“现在该用哪个”

因此当前原型已经改成：

- 常规状态下不再发射攻击型剑阵
- 保留连续变形、飞剑环绕、飞剑库存可读性
- 三形态的攻击兑现改为主要发生在 `强化剑技` 窗口内
- 日常主动消耗位重新集中给 `御剑`

这次调整的目标不是削掉剑阵存在感，而是把循环重新理顺：

- 平时：`左键近战 / 弹反`
- 平时：`右键御剑`
- 爆发：`Space 开启持续飞剑爆发`

这一版解决的是“常规剑阵和御剑抢地位”的结构问题。

下一轮实机要重点观察的是：

- 现在日常战斗里，`御剑` 是否终于重新拥有清晰地位
- 常规状态因为没有攻击型剑阵，循环是否反而更自然
- 强化剑阵是否因为被集中到爆发层而显得更有价值
- 连续变形在不承担常规攻击职责后，是否仍然保留了足够的签名体验

## 6. 当前最重要的实现与验证边界

为了避免把系统重新打散，当前阶段采用如下边界：

- 保留连续变形
- 保留距离决定主导阵型
- 优先调行为层与 sortie profile
- 尽量不改阵型几何大框架
- 只有当行为层已经无法继续表达形态身份时，才考虑更高冲击的结构重做

## 7. 下一阶段继续验证时要重点观察什么

后续每轮实机反馈，建议优先回答这几个问题：

1. 哪个形态现在最可信，为什么可信？
2. 哪个形态仍然会在关键时刻“看起来对，但打起来不对”？
3. 当前持续射击的控制平滑，是变稳了，还是开始有点拖手？
4. 三形态现在的近中远分工，是否已经能在普通战斗里被自然感知到？
5. 如果再做回收逻辑差异化，最优先该补的是哪一种阵型的身份？
6. 现在技能放不出来时，玩家能不能立刻知道是 `剑意不足`、`飞剑未回收` 还是 `剑丸不足`？
7. 剑丸掉落现在是否回到了“爆发资源”的感觉，而不是随手过剩？
8. 普通攻击弹反现在是更爽了，还是已经快到开始影响读感？
9. 取消常规攻击型剑阵后，`御剑` 是否重新变成了你会下意识使用的日常工具？
10. 爆发剑阵现在是否比之前更像“真正的爆发层”，而不是常规玩法的上位替代？

## 8. 当前结论

截至本轮讨论，可以先把剑阵系统理解为：

- 强化剑技可靠性问题：已基本解决
- 环阵近程身份问题：已基本建立
- 常规攻击型剑阵：已从日常循环中移除
- 连续变形与稳定控制的矛盾：已找到可行折中
- 三形态回收语义差异：基础版已进入当前实现，后续主要是继续抛光身份
- 技能不可释放时的反馈：已进入当前实现，后续主要验证是否还要补音效或更强表现
- 剑丸经济节奏：已开始收紧，需要继续验证强化窗口是否健康
- 普通攻击弹反观感：已提升速度，需要继续验证是否达到理想的反杀读感
- `御剑` 与 `剑阵` 的日常地位冲突：已通过“取消常规攻击型剑阵”进入结构修正阶段

因此，当前最合适的推进方式不是重新推翻框架，而是：

**在保留连续变形和职责分工的前提下，继续用小步验证的方式，把“日常御剑 + 爆发剑阵”这套新循环做扎实，再继续抛光强化窗口内的三形态身份。**

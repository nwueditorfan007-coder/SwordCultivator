# 飞剑锻造与剑相系统讨论记录

日期：2026-05-07

本文记录本日关于飞剑锻造、剑灵、材料属性和战斗形态的讨论过程。它是讨论记录，不是最终实现文档。新会话若要直接实现四把验证剑，请读 `docs/internal/SWORD_FORGING_FOUR_SWORDS_HANDOFF_2026-05-07.md`。

## 1. 起点问题

最初讨论的是游戏缺少令人惊艳的新颖机制。经过多轮排除后，结论逐渐从“新增外层玩法”转向当前战斗里最有辨识度的东西：

- 鼠标御剑
- 12 把可见飞剑库存
- 鼠标距离驱动的环阵、扇阵、贯穿阵连续变形

玩家提出希望引入“剑灵”，把飞剑从工具变成伙伴，并进一步提出通过锻造材料产生不同手感。

## 2. 已否定或收窄的方向

### 2.1 不做单纯语音释放

语音释放如果只是把按键变成口令，会变成噱头。

有价值的是“宣告、响应、结算”的仪式感，但这不适合先作为当前锻造系统的核心。

### 2.2 不做剑灵抢操作

剑灵不应该替玩家玩游戏。尤其不能：

- 自动替玩家选择环/扇/贯
- 自动处理所有目标
- 用自动索敌掩盖玩家操作

剑灵以后应该作为“执行倾向”或“行为偏好”，而不是替代玩家判断。

### 2.3 不把基础手感做成养成解锁

基础响应必须一开始可靠。不能让玩家前期觉得剑迟钝，然后靠升级解锁正常手感。

材料和锻造可以改变节奏、承诺感、回收收益和剑阵组织方式，但不能让玩家觉得输入不被信任。

### 2.4 不做泛用动作游戏假设

讨论中曾错误引入“闪避”“环阵缺口”“自动选择贯穿线”等通用动作游戏设计语言。后来明确修正：

- 当前游戏没有闪避。
- 环阵没有缺口。
- 阵型由鼠标距离决定。
- 贯穿阵不能由系统自动替玩家选，否则玩家不需要操作。

后续设计必须从当前项目实际输入出发。

## 3. 当前项目边界

以 `docs/project/CURRENT_PROJECT_DESIGN.md` 为准：

- `左键点按`：近战挥剑，可弹反。
- `左键长按约 0.10s`：剑阵持续输出。
- `右键点按`：御剑点刺。
- `右键长按`：御剑连斩，松开回收。
- 剑阵由鼠标距离连续决定：环阵、扇阵、贯穿阵。
- 当前剑阵有 12 把可见飞剑。
- 御剑是高辨识度主操作。
- 剑阵连续变形是项目签名体验，需要保护。

## 4. 关于轻剑和重剑的关键判断

早期曾提出“轻灵和重势可以主副共存”，用户指出这是错误的。

最终修正为：

**轻灵和重势是一条轴的两端，不能同时作为高值主属性。**

正确理解：

- 轻灵：右键长按适合连续滑动追杀。
- 重势：右键长按适合压剑线、蓄势、松开重斩。

它们对应不同操作语法，不是两个可以同时堆高的词条。

## 5. 参考《铸剑吟》的正确方式

讨论中参考了《铸剑吟 / Bladesong》的锻造思路。真正值得参考的不是复杂属性数量，而是因果链：

**玩家改变剑的物理结构，系统自然推导出剑的属性和手感。**

因此不应该让玩家直接选择：

- 轻灵
- 重势
- 归锋
- 分光

这些应该是结果，而不是锻造面板上的原始属性。

## 6. 精简后的锻造因果链

最终建议：

玩家锻造时只关心 4 类输入：

| 锻造项 | 玩家理解 | 系统影响 |
| --- | --- | --- |
| 主材 | 材质是什么 | 密度、硬度、韧性、导灵性 |
| 剑身 | 长、宽、厚、薄、窄 | 重量、刃压、转动惯量 |
| 配重 | 剑首、剑格、重心 | 出剑强还是收剑强 |
| 灵槽 | 导灵槽走向 | 凝锋、分光、回流 |

系统内部可以推导：

- 总重
- 重心
- 转动惯量
- 刃压
- 刚性
- 导灵结构

但玩家最终只需要看 3 条核心剑相轴。

## 7. 三条核心剑相轴

后续讨论修正了一个重要问题：如果三条剑相只影响右键或少数剑阵动作，玩家会觉得“某个技能变了”，而不是“整把剑变了”。

因此最终原则改为：

**每条剑相都必须影响 `普攻 / 御剑点刺 / 御剑斩击 / 环阵 / 扇阵 / 贯穿阵` 六个动作。**

实现层应采用轴向组合：

`最终动作 = 基础动作 + 剑重 modifier + 剑衡 modifier + 剑路 modifier`

四把验证剑只是三条轴的组合样本，不应为每把剑硬写一套不可扩展的独立技能。

### 7.1 剑重：轻灵 / 重势

决定节奏、惯性和承诺感。

| 动作 | 轻灵 | 重势 |
| --- | --- | --- |
| 普攻 | 快速斩，后摇短，适合连续贴身补刀和弹反后追击 | 重斩，后摇更有承诺，但击退、破势、弹反反馈更强 |
| 御剑点刺 | 快速点刺，快速回收，适合切 Silk 和补残血 | 重刺钉点，命中有停顿、打断、震退 |
| 御剑斩击 | 保留当前右键长按滑动追杀，飞剑贴鼠标 | 右键长按改为压剑线，按住蓄势，松开重斩 |
| 环阵 | 高速近身游剑，偏切贴脸敌人 | 近身剑罩/剑墙，偏清弹、抗压、反震 |
| 扇阵 | 高频薄剑幕，连续扫 | 低频厚横扫，推开一侧战线 |
| 贯穿阵 | 逐把细刺，快而连续 | 12 剑合成巨剑劈下或贯下 |

### 7.2 剑衡：出势 / 归锋

决定收益发生在出剑时，还是回收时。

| 动作 | 出势 | 归锋 |
| --- | --- | --- |
| 普攻 | 向前斩出的第一段更强 | 收剑回身时带二段回刃或回身切割 |
| 御剑点刺 | 飞出去命中最强 | 点刺只是挂线，回收路径更强 |
| 御剑斩击 | 按住期间持续切割更强 | 按住期间偏布线，松开回收爆发 |
| 环阵 | 剑阵向外顶开压力 | 剑阵向内收束，回卷清周身 |
| 扇阵 | 扇形扫出去的首波强 | 扇阵扫出后回卷二段强 |
| 贯穿阵 | 贯出去那一下强 | 穿出后归线斩强 |

### 7.3 剑路：凝锋 / 分光

决定力量是集中成一线，还是分成多线。

| 动作 | 凝锋 | 分光 |
| --- | --- | --- |
| 普攻 | 单条清晰剑弧，窄但深 | 多道浅剑影，宽但单段轻 |
| 御剑点刺 | 单点高质量刺击 | 主刺旁带短延迟分光 |
| 御剑斩击 | 一条干净切线，适合精准切 Silk | 鼠标轨迹有延迟复刻剑光 |
| 环阵 | 紧密单环，像一条切割带 | 内外双层薄环，不出现缺口 |
| 扇阵 | 扇面更窄、更密、更狠 | 扇面更宽，多线覆盖 |
| 贯穿阵 | 单线贯穿或巨剑贯穿 | 两到三条并行贯穿线 |

## 8. 四把验证剑

为避免一开始做完整锻造系统，建议先做四把固定测试剑。

### 8.1 风砂银分光剑

- 剑重：轻灵
- 剑衡：出势
- 剑路：分光

关键词：快速多影普攻、滑动追杀、多线剑幕、并行细线贯穿。

### 8.2 寒玉归锋剑

- 剑重：轻灵
- 剑衡：归锋
- 剑路：凝锋

关键词：轻快单线普攻、精准点刺、收剑爆点、凝线回斩。

### 8.3 玄铁巨阙剑

- 剑重：重势
- 剑衡：出势
- 剑路：凝锋

关键词：厚重单线普攻、重刺钉点、压剑线、剑罩、巨剑劈下。

### 8.4 星陨回渊剑

- 剑重：重势
- 剑衡：归锋
- 剑路：分光

关键词：重斩多影普攻、重刺回收、压剑线、多线回卷、回渊收束。

## 9. 剑灵性格暂留接口

本日没有展开剑灵性格对战斗形态的完整影响。当前只留下边界：

- 锻造决定剑怎么动。
- 剑灵决定它在这些动作里偏向什么行为。
- 剑灵不能重写锻造形态。
- 剑灵不能替玩家选择阵型。

可预留事件：

- 右键点按命中
- 右键长按持续命中
- 松开右键回收
- 环阵发射/维持
- 扇阵发射/回收
- 贯穿阵命中
- 切断 Silk
- 清掉子弹
- 剑意低
- 玩家低血

## 10. 后续实现优先级

推荐实现顺序：

1. 先实现 `轻灵` 与 `重势` 对六个动作的差异，尤其是御剑斩击和环阵。
2. 再实现 `出势` 与 `归锋` 对六个动作的收益时机差异，尤其是普攻回刃和御剑回收。
3. 最后实现 `凝锋` 与 `分光` 对六个动作的形态差异，尤其是扇阵和贯穿阵。
4. 再组合出四把验证剑。
5. 只在这些成立后，再讨论剑灵性格。

最小验证可以先做两把：

- 风砂银分光剑
- 玄铁巨阙剑

如果这两把无法让玩家清楚感到“轻剑追杀、重剑压阵”，后续系统不应继续扩。

## 11. 核心设计结论

本日最终结论：

**锻造系统的重点不是给飞剑加属性，而是让玩家相信：我改了剑的身体，所以它自然以不同方式御剑、成阵、出锋、归锋。**

这个差异必须贯穿整套输入。只改变右键或只改变某个剑阵，都不足以支撑“这把剑的身体不一样”的代入感。

这比“材料给数值，性格给特效”更贴近修仙炼剑，也更适合当前项目的鼠标御剑和连续剑阵变形。

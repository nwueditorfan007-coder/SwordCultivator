# 剑修 Demo 主视觉母版任务包 V1

## 1. 任务目的

本任务包用于产出一张可以作为整个 demo 视觉母版的主视觉英雄镜头。

这张图的职责不是宣传海报，而是作为后续所有内容的母版依据：

- 背景底图方向
- HUD 风格语言
- 飞剑亮度规则
- 红色敌方威胁语言
- 书法 / 阵图 / 金线的密度尺度

如果这张图没有立住，后续所有零件都会失去统一标准。


## 2. 母版定位

一句话命题：

`玄夜天幕中的剑修法阵战斗。`

必须同时满足：

- 有东方修仙气质
- 有单屏战斗舞台感
- 有高级克制感
- 有清晰的飞剑主角地位
- 有明确的敌我识别

不是：

- 普通古风插画
- 大场景角色海报
- 页游式华丽堆料
- 赛博 UI 拼贴图


## 3. 构图规格

建议先统一为：

- 画幅：`16:9`
- 推荐尺寸：`1280x720` 或更高
- 镜头语义：俯视角单屏战斗

构图要求：

1. 画面必须存在明确阵心。
2. 玩家与飞剑必须构成唯一视觉焦点。
3. 飞剑主要攻击方向必须清晰可读。
4. 背景必须为战斗让出空间，不可抢主体。
5. HUD 或法器边框可以轻度入画，但不能比飞剑更抢眼。


## 4. 画面内容清单

必须出现的内容：

- 中心阵心
- 剑修或剑修存在的视觉暗示
- `8 - 14` 把飞剑
- 一条最强主剑势
- 若干辅助飞剑轨迹
- 至少 `3 - 5` 个敌方威胁符标
- 背景书法残字
- 轻量金线阵图
- 深色天幕或法阵底

可以出现但必须克制的内容：

- 云纹
- 星点
- 侧边箴言
- 顶部标题
- 极弱 HUD 装饰

不要出现的内容：

- 大面积角色服装细节
- 写实山石建筑
- 复杂怪物立绘
- 高密度场景道具


## 5. 风格规则

## 5.1 颜色

### 基底

- 深蓝黑
- 暗青灰

### 玩家 / 飞剑

- 冷白核心
- 青蓝边缘

### 秩序与法器

- 古金细线

### 敌方威胁

- 绯红
- 暗红

### 禁止项

- 大面积紫色主导
- 彩虹色能量
- 高饱和蓝绿乱跳

## 5.2 亮度

亮度优先级必须是：

1. 飞剑本体与核心剑势
2. 短促宣告 / 命中高点
3. 少量 HUD 核心提示
4. 敌方威胁符标
5. 阵图与背景

金色不能比飞剑更亮。

## 5.3 材质感

建议画面质感为：

- 深、薄、静的背景
- 锐、脆、快的飞剑
- 轻、细、准的金线
- 邪术感强但面积小的红色节点


## 6. 主提示词

```text
为一款中国修仙题材的俯视角单屏动作游戏设计一张主视觉母版图。

画面为玄夜天幕中的法阵战场，整体以深蓝黑为主。中央有一处发光阵心，一名剑修或其存在感位于阵心附近，十余把飞剑围绕并向外爆射。飞剑呈冷白核心、青蓝边缘，纤长、锐利、神性、速度感强，其中一条主剑势最强、最亮、最具支配性，其余飞剑形成辅助轨迹和阵列变化。

战场背景不是写实地面，而是活着的阵图：极淡金线同心环、弧轨、星点、水墨云纹、书法残字、东方神秘感。敌方只以少量绯红色术式符标和危险节点存在，不做复杂怪物立绘。画面边缘可轻度出现 HUD 或法器边框感装饰，如顶部题字、左右箴言、底部技能盘暗示，但必须很克制。

风格关键词：玄夜天幕，星图法阵，水墨余痕，古金秩序线，冷白青蓝飞剑，绯红邪术威胁，高级感，克制，留白，中心构图，单屏战斗海报感，东方宇宙感。

硬性要求：
1. 飞剑必须是最高亮度主体。
2. 背景必须暗、静、薄。
3. 金色只做秩序和法器边框，不与飞剑争主角。
4. 红色只用于敌方威胁和机制标记。
5. 中央战斗区要有明显留白，不可堆满装饰。

禁止项：
- 赛博霓虹
- 页游仙侠大翅膀
- 写实地砖或山门场景
- 复杂怪物特写
- 大面积高饱和紫色
- 装饰过满导致画面脏乱

输出目标：一张可以作为整个 demo 视觉基准的主视觉母版图。
```


## 7. 变体方向

建议一次生成 `3` 个方向，不要只抽一张。

### 方向 A：阵盘图谱型

- 更克制
- 更偏 `图4 / 图5`
- 阵图更清晰
- HUD 入画更强

### 方向 B：玄夜星图型

- 更空灵
- 更偏 `图1 / 图6`
- 星轨与书法感更强
- 背景更 painterly

### 方向 C：中间平衡型

- 用于最终最可能落地
- 背景克制
- 战斗信息清晰
- 气质仍高级


## 8. 评审标准

先看这五个问题：

1. 缩小后第一眼看到的是不是飞剑？
2. 画面中央是不是像战斗舞台，而不是插画背景？
3. 红色威胁是不是清楚但不抢主角？
4. HUD / 金线装饰是不是法器感，而不是普通 UI？
5. 去掉文字后，这张图是否仍然成立？

如果其中任意两项不成立，这张母版不能进入后续扩展。


## 9. 二轮修订指令

如果第一轮结果不理想，优先按以下方式单变量修订：

### 修订 A：压背景

```text
保持整体构图不变，仅降低背景纹样密度和亮度，让中心战斗区更干净。
```

### 修订 B：提飞剑

```text
保持构图不变，仅增强飞剑的主角地位，让飞剑更锐、更亮、更统一。
```

### 修订 C：压金色

```text
保持构图不变，仅减少古金装饰的亮度和面积，让其更像秩序框架而不是主体。
```

### 修订 D：压 UI 感

```text
保留法器边框语义，但减少现代 HUD 面板感，让装饰更融入战场。
```


## 10. 落地后的用途

这张母版确认后，应直接反推：

- 背景底图规则
- HUD 样式
- 飞剑贴图方向
- 敌方符标系统
- Godot 合成参数目标

它不是“好看一张图”就结束，而是整个视觉生产线的母体。

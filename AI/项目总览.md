# 剑修弹幕 Demo 项目总览

## 项目目标

项目暂定名：`剑修弹幕 Demo`

这是一个基于 Godot 开发的俯视角动作弹幕原型。当前目标不是做完整游戏，而是在最短时间内验证以下三件事：

1. 近战弹反是否足够爽。
2. 远程御剑切割是否足够顺手。
3. 双形态切换是否能形成自然战斗节奏。

只要这三个问题得到正向验证，项目就具备继续扩展的价值。

## 一句话卖点

在弹幕中切换护体剑与御剑术：近战弹反子弹，远程操控飞剑切开敌人。

## 玩家核心循环

1. 敌人入场并发射弹幕。
2. 玩家移动走位。
3. 高压时切近战形态处理弹幕。
4. 通过斩碎或弹反获得生存空间和剑意。
5. 找到空档后切远程形态追击敌人。
6. 使用飞剑快速切割、收割目标。
7. 击败敌人并进入更高压力波次。

目标是让玩家在 30 秒内理解，在 60 秒内感受到核心爽点。

## Demo 范围

### 必做

- 1 个可操控角色
- 近战 / 远程双形态
- WASD 移动
- 左键攻击，右键切换形态
- Space 冲刺，Q 绝招
- 普通弹 / 危险弹两种弹幕
- 3 类敌人
- 3 分钟单局关卡
- 生命条 / 剑意条 / 当前形态 UI

### 暂不制作

- 多角色
- 长线成长树
- 装备和词条
- 剧情系统
- 大地图
- 复杂 Boss

## 战斗设计原则

### 近战形态

- 用于防守、生存、处理弹幕
- 左键触发扇形斩击
- 普通子弹可被斩碎或弹反
- 成功处理弹幕可积累剑意

### 远程形态

- 用于输出、追击、斩杀
- 飞剑脱离角色并受鼠标控制
- 接触敌人造成持续切割伤害
- 输出高，但生存能力弱

### 切换机制

- 近战用于活下来
- 远程用于更高效杀敌
- 切换本身应带有效果和反馈

## 验收标准

### 玩家理解

新玩家在 1 分钟内能理解：

- 近战用于防守和弹反
- 远程用于切割和输出
- 切换是核心战斗节奏

### 体验成立

玩家至少能明确感受到一次：

- 弹反很爽
- 飞剑切人很舒服
- 形态切换带来节奏变化

### 工程标准

- Demo 可稳定运行
- 核心模块结构清晰
- 代码可继续扩展
- 便于下一轮加入反馈、美术、波次和成长

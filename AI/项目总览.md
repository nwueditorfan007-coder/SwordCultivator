# 剑修弹幕 Demo 项目总览

## 项目目标

这是一个基于 Godot 的俯视角动作弹幕原型。

当前最值得验证的不是“系统够不够多”，而是下面四件事能不能一起成立：

1. 近战弹反是否足够可信。
2. 右键御剑带来的子弹时间与切丝体验是否足够鲜明。
3. 连续变形剑阵是否真的能形成近中远三种稳定战术语义。
4. `Boss / Puppet / Silk` 机制能否把战斗从清怪推进到“切机制开窗口”。

## 一句话卖点

在弹幕压力里用近战弹反保命，用御剑切丝开窗，再用连续变形剑阵把输出兑现出来。

## 当前实际核心循环

1. 敌人入场并制造弹幕压力。
2. 玩家用走位和左键近战处理贴身威胁。
3. 通过近战命中、弹反和击杀积累剑意。
4. 用右键御剑点刺或连斩处理高价值目标、切 `Silk`、打开时间差。
5. 用左键长按直接维持剑阵输出，根据鼠标距离在环阵 / 扇阵 / 贯穿阵之间切换。
6. 在 Boss 战中优先处理 `Puppet / Silk`，创造破绽窗口后再补伤害。

## 当前主场景的真实操作

- `WASD`：移动
- `左键点按`：近战挥剑
- `左键长按`：维持剑阵持续输出
- `右键点按`：御剑点刺
- `右键长按`：御剑连斩

注意：

- 当前主场景没有“右键切换近战/远程形态”的独立按钮
- 当前主场景也没有 `Space` 冲刺或强化剑技

## 当前战斗设计判断

当前版本最稳定的战斗语言已经不是早期那种“近战 / 远程双形态切换”，而是：

- `近战`：生存、弹反、建节奏
- `御剑`：时停、切机制、高价值处理
- `剑阵`：常规持续输出与距离分工
- `Boss 机制`：通过切丝与破绽控制推进

所以现在更准确的理解应是：

**这是一个以近战弹反、御剑时停、连续变形剑阵和切丝开窗为核心的动作弹幕原型。**

## 当前最需要记住的实现事实

- 旧版“冻结弹幕 -> 主动吸收 -> 再发射”已经不再是主循环
- 当前剑阵直接由常驻飞剑库存构成，不需要先吸收弹幕
- 当前御剑不消耗剑意
- 当前剑阵是日常可用的持续输出位，不是只在强化窗口里才出现的爆发层

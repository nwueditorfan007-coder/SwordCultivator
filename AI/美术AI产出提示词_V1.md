# 剑修 Demo 美术 AI 产出提示词 V1

## 1. 文档目的

本文件用于让后续 AI 会话可以直接开始产出：

- 主视觉
- HUD 方向稿
- 背景底图
- 敌方符标
- 飞剑设计
- 阵图装饰件

默认风格基线以：

- `docs/systems/DEMO_ART_DIRECTION_AND_AI_PIPELINE_V1.md`
- `docs/systems/COMBAT_VFX_ART_FRAMEWORK_V1.md`

为准。


## 2. 总提示词骨架

所有图像类任务都建议遵循以下结构：

1. 先明确画面类型
2. 再明确风格关键词
3. 再明确禁止项
4. 再明确构图与亮度规则
5. 再明确输出用途

推荐骨架：

```text
为一款中国修仙题材的俯视角单屏动作游戏设计【具体产物】。

视觉方向：玄夜天幕、星图法阵、水墨余痕、古金秩序线、冷白青蓝飞剑、绯红敌方威胁、克制而高级、留白充足、强中心构图、战场像活着的阵图。

核心要求：
1. 飞剑必须是最高亮度主体。
2. 背景保持深蓝黑、静、薄，不抢战斗主体。
3. UI 或装饰必须像法器的一部分，而不是现代面板。
4. 金色用于秩序和边框，不与飞剑争主角。
5. 红色只用于敌方威胁与机制标记。

禁止项：
- 赛博霓虹
- 页游仙侠大翅膀
- 花哨彩虹渐变
- 厚重金属面板 UI
- 写实场景地砖
- 大面积高饱和紫色
- 过度拥挤、没有留白

输出目标：【具体说明】
```


## 3. 主视觉英雄镜头

## 3.1 用途

用于先定整套 demo 的母版气质。

## 3.2 提示词

```text
为一款中国修仙题材的俯视角单屏动作游戏设计一张主视觉英雄镜头。

画面内容：黑蓝色夜幕法阵战场中央，一名剑修立于阵心，十余把飞剑围绕并向外爆射，飞剑呈冷白与青蓝发光，轨迹锐利、纤长、克制。战场外围是深色水墨云纹、星点、淡书法残字和极细金线阵图。敌方威胁以绯红色符标和少量红色术式节点存在，不做复杂怪物立绘。HUD 与战场一体化，顶部和侧边带有古金色细边、道法题字、资源条与技能盘装饰，但整体仍然轻、薄、克制。

风格关键词：玄夜天幕，星图法阵，水墨余痕，古金秩序线，冷白青蓝飞剑，绯红术式威胁，高级感，留白，中心构图，单屏战斗海报感，东方神秘感。

构图要求：
1. 玩家与飞剑是唯一绝对视觉中心。
2. 飞剑亮度最高，背景压暗。
3. 画面必须有阵心、轨道弧线和明确攻击方向。
4. 大书法字只作为氛围背景，不抢主体。
5. 允许少量粒子和尘埃感，但不可脏乱。

禁止项：
- 角色特写占画面太大
- 背景做成真实山水场景
- UI 面板化
- 红蓝对冲过满
- 每个角落都塞满装饰

输出目标：一张可作为整套 demo 视觉母版的高完成度概念图。
```

## 3.3 验收点

- 缩小后第一眼先看到飞剑，而不是 UI
- 去掉文字后仍然成立
- 替换敌人后风格仍能保留


## 4. HUD 方向稿

## 4.1 用途

用于替换当前功能型 HUD。

## 4.2 提示词

```text
为一款中国修仙题材的俯视角动作游戏设计 3 套 HUD 风格稿，统一世界观为玄夜法阵剑修。

HUD 内容包含：
- 左上生命与剑意
- 顶部中央标题区域
- 右上模式或状态图标
- 底部技能盘和按键提示
- 两侧竖排题字或道法短句

视觉方向：古金细线、淡发光、法器感、阵图感、轻量、薄、克制，不做厚面板。资源条应像嵌在阵图边框中的法器槽，不是现代 RPG 长条。技能图标应像刻在法阵上的符印，不是扁平 app icon。

要求：
1. 信息层级清晰，数值小字必须可读。
2. 装饰要有东方仪式感，但不能过重。
3. 中心战场区域尽量让开。
4. 提供三种密度：极简版、平衡版、稍华丽版。

禁止项：
- 科幻仪表盘
- 厚重实体框
- 大面积纯金属质感
- 紫色主导
- 复杂卡牌式技能窗

输出目标：三套同风格不同密度的 HUD 概念稿。
```

## 4.3 验收点

- 不遮战场
- 资源条看起来像法器，不像网页进度条
- 按键提示和技能图标不违和


## 5. 背景底图

## 5.1 用途

用于 `Scene Base`。

## 5.2 提示词

```text
为一款俯视角单屏修仙动作游戏设计战场背景底图。

画面目标：不是具体地面场景，而是一张玄夜法阵舞台。主体是深蓝黑底色、极弱星点、淡金色同心环和弧线、边缘水墨云气、隐约书法残字、极淡纹样。中心区域为战斗预留留白，不放复杂图案。整体要高级、安静、神秘，有东方宇宙感。

要求：
1. 中央留白明显，方便叠加战斗。
2. 图案密度从中心向边缘增加。
3. 所有纹样都要轻、薄、暗。
4. 可提供 2 个方向：更星图、更水墨。

禁止项：
- 写实地板
- 明显透视场景
- 大面积高亮纹路
- 背景比角色还亮

输出目标：适合作为战斗底层的背景概念图或可切层底图。
```

## 5.3 验收点

- 单独看有气质
- 叠 HUD 和战斗后不脏
- 中心能承受高亮飞剑


## 6. 敌方符标系统

## 6.1 用途

用于替代复杂敌人立绘，先建立威胁语言。

## 6.2 提示词

```text
为一款修仙题材俯视角动作游戏设计一组敌方威胁符标系统。

需求内容：
- 普通远程目标符标
- 高威胁狙击目标符标
- Boss 机制锚点
- 丝线节点
- 锁定 / 警告 / 蓄力状态变化

风格方向：绯红、暗红、白热核心、术式感、几何化、克制、危险、精确。符标应更像邪术印记与阵法节点，而不是科幻准星。允许少量锐利分叉、菱形、圆环、十字放射，但必须统一一套语法。

要求：
1. 平时亮度低于飞剑。
2. 警告时可以短暂升亮。
3. 轮廓必须清晰，缩小后仍可辨认。
4. 同一套系统里要有普通、强化、蓄力、危险升级四个层级。

禁止项：
- 赛博 UI
- 过多机械结构
- 纹样过于写实
- 红色泛滥成霓虹灯

输出目标：一整套统一语法的敌方符标设计板。
```

## 6.3 验收点

- 一眼能区分友方和敌方
- 小尺寸仍有辨识度
- 不需要怪物本体也能先成立


## 7. 飞剑本体探索

## 7.1 用途

用于后续飞剑贴图、轮廓、拖尾风格统一。

## 7.2 提示词

```text
为一款中国修仙题材俯视角动作游戏设计一组飞剑本体探索图。

目标：飞剑需表现为神兵、法器、速度、锋锐，不走厚重写实武器路线。整体以纤长比例、冷白核心、青蓝边缘、少量古金结构点缀为主。允许有道纹、剑格、尾端灵气结构，但整体必须利落，不可复杂到缩小后看不清。

要求：
1. 先出 3 个方向：极简神兵型、古雅法器型、灵纹强化型。
2. 每个方向都给出俯视远看效果的考虑。
3. 飞剑缩小成战斗尺寸后仍有轮廓。
4. 优先考虑拖尾和高亮核心如何结合。

禁止项：
- 重兵器
- 西式骑士剑
- 夸张大护手
- 细节堆满导致远看糊成一团

输出目标：适合战斗中高频出现的飞剑设计探索板。
```

## 7.3 验收点

- 缩到战斗尺寸仍锐
- 不像普通武器图鉴
- 和 HUD / 背景放一起不违和


## 8. 装饰件与阵图零件

## 8.1 用途

用于 UI Ornament 和 Tactical Glyph。

## 8.2 提示词

```text
为一款修仙题材俯视角动作游戏设计一组可复用的阵图装饰件与古金边框零件。

内容包括：
- 四角角饰
- 顶部标题边饰
- 圆环分割线
- 资源条边框
- 底部技能盘底座
- 侧边细长竖排装饰
- 微型印章和小型符纹

视觉方向：古金细线、阵图法器感、精密但克制、东方神秘感、轻薄、高级。更像法阵和器物铭纹，不像宫廷花纹。

要求：
1. 尽量模块化，方便后续拼装。
2. 线条细，亮度低于主战斗元素。
3. 可同时适配圆形、横条、竖条三种区域。

禁止项：
- 复杂浮雕
- 厚重金属块
- 古风婚庆纹样
- 装饰密度过满

输出目标：可拆分复用的一套装饰件设计板。
```


## 9. AI 代码与实现辅助提示

## 9.1 HUD 落地

```text
基于 Godot 4 的 2D 单屏动作项目，为当前 HUD 提出一版可落地的重构方案。

要求：
1. 保留现有信息结构，不新增玩法信息。
2. 将 HUD 视觉改为“玄夜法器阵图”风格。
3. 输出结构包括：节点建议、绘制层级、贴图资源分类、可代码绘制部分、必须美术资产化部分。
4. 不要给泛泛建议，要面向当前项目的 `Node2D + draw_*` 路线。
```

## 9.2 后处理草案

```text
基于 Godot 4 的 2D 单屏战斗 demo，设计一套轻量 composite / post-processing 方案。

目标风格：深蓝黑背景、冷白飞剑、古金 HUD、绯红敌方符标，要求高级、克制、不糊。

请输出：
1. 适合当前 demo 的后处理项优先级
2. 哪些可以先不做
3. 如果使用 SubViewport 或 shader，建议的最小方案
4. 需要重点避免的可读性风险
```


## 10. 使用规则

- 每次只让 AI 解决一个明确产物
- 每次保留禁止项
- 每次都写清楚“飞剑必须是最高亮度主体”
- 每轮只固定一个变化变量
- 不满意先做定向修订，不要完全重抽

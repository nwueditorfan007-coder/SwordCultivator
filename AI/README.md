# AI 文档目录

这个目录用于存放所有给 AI 阅读的项目文档，目标是让任何一个新的 AI 会话都能快速理解项目目标、当前阶段、约束和下一步工作。

## 当前文档

- `项目总览.md`：项目定位、核心玩法、Demo 范围、验收目标。
- `开发计划.md`：当前阶段的执行顺序、验证目标、实现边界和下一轮试玩问题。
- `AI协作规范.md`：AI 在本项目中的职责边界、产出格式和协作规则。
- `启动清单.md`：第一轮开发启动期的任务清单，现主要作为历史参考。

## 使用规则

1. 新开 AI 会话时，优先阅读本目录下的文档。
2. 项目范围、里程碑、节点结构、命名规范发生变化时，优先更新这里。
3. 设计结论写进文档，不只留在聊天记录里。
4. 每完成一个阶段，补充实际结果、偏差和下一步计划。

## 建议后续补充的文档

- `任务看板.md`：按进行中 / 待办 / 已完成维护任务状态。
- `版本记录.md`：记录每轮迭代新增内容和回归风险。
- `数值草案.md`：敌人、弹幕、剑意、伤害等参数表。
- `场景结构.md`：Godot 场景树、节点约定、资源路径说明。
